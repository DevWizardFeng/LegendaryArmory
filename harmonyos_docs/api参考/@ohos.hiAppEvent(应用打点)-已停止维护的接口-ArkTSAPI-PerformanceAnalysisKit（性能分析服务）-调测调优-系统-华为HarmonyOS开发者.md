class="support-device-item">Wearable</span></div><p>开发者在使用应用事件打点功能前，需要首先了解应用事件相关的参数规格定义。</p> <p><strong>事件名称</strong></p> <p>事件名称为string类型，字符串非空且长度在48个字符以内，首字符必须为字母字符或$字符，中间字符必须为数字字符、字母字符或下划线字符，结尾字符必须为数字字符或字母字符。</p> <p><strong>事件类型</strong></p> <p>事件类型为<a href="/consumer/cn/doc/harmonyos-references/js-apis-hiappevent#eventtype">EventType</a>枚举类型。</p> <p><strong>事件参数</strong></p> <p>事件参数为object类型，key为事件的参数名称，value为事件的参数值，其规格定义如下：</p> <ul><li>参数名为string类型，字符串非空且长度在32个字符以内，首字符必须为字母字符或$字符，中间字符必须为数字字符、字母字符或下划线字符，结尾字符必须为数字字符或字母字符；</li><li>参数值支持string、number、boolean、数组类型；</li><li>参数值为string类型时，其长度需在8*1024个字符以内，超出会做丢弃处理；</li><li>参数值为number类型时，其取值需在Number.MIN_SAFE_INTEGER~Number.MAX_SAFE_INTEGER范围内，超出可能会产生不确定值；</li><li>参数值为数组类型时，数组中的元素类型只能全为string、number、boolean中的一种，且元素个数需在100以内，超出会做丢弃处理；</li><li>参数个数需在32以内，超出的参数会做丢弃处理。</li></ul> <p><strong>事件回调</strong></p> <p>开发者在调用事件打点方法后，可以在回调函数中对打点返回值进行处理，当前支持callback形式和Promise形式的回调，其返回值规格定义如下：</p> <ul><li>返回值为0，表示事件校验成功，将事件直接落盘到事件文件；</li><li>返回值大于0，表示事件校验存在异常参数，在忽略异常参数后将事件落盘到事件文件；</li><li>返回值小于0，表示事件校验失败，不将事件落盘到事件文件。</li></ul> </div>  <div class="tiledSection"><h2 id="hiappeventwrite"
