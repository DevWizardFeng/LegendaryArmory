class="anchor-icon anchor-icon-link" anchorid="命名规则" tips="复制节点链接"></i></h3>          <p><strong>命名形式：</strong></p>     <p>标准URI定义主要由以下三个部分组成：</p>     <p>[scheme:]scheme-specific-part[#fragment]。</p>     <p>细化URI格式，可以将其分为：</p>     <p>[scheme:][//authority][path][?query][#fragment]。</p>     <p>将URI格式进一步细化，可以分为：</p>     <p>[scheme:][//[user-info@]host[:port]][path][?query][#fragment]。</p>     <ul>      <li>scheme: 协议名，与scheme-specific-part以:进行分隔，包含scheme部分的URI为绝对URI，不包含scheme部分的URI为相对URI，根据需要填写。例如http、https、ftp、datashare等。</li>      <li>scheme-specific-part: URI的特定解码方案特定部分，位于[scheme:]和[#fragment]之间由[//][authority][path][?query]组成，此部分以/开头的为分层URI，不以/开头的为不透明URI，根据需要填写。       <ul>        <li>authority: URI的解码权限组件部分。由[userinfo@]host[:port]组成，根据需要填写。         <ul>          <li>userinfo: 用户信息，与host通过@进行分隔，根据需要填写。</li>          <li>host: 服务器的主机名部分，当authority存在时，此项必填。</li>          <li>port: 服务器端口，默认值为-1。根据需要填写。</li>         </ul></li>        <li>path: 路径信息，位于host与query之间以 / 进行分割，根据需要填写。</li>        <li>query: 查询部分，位于path和fragment之间，以 ? 开头的键值对格式，以&amp;分割键值对，以=分割键值，根据需要填写。</li>       </ul></li>      <li>fragment: 片段部分，以#与scheme-specific-part进行分隔，根据需要填写。</li>     </ul>     <p><strong>URI示例：</strong></p>     <div _ngcontent-eyd-c106="" class="highlight-div"><div _ngcontent-eyd-c106=""
