class="support-device-item">TV</span><span class="support-device-item">Wearable</span></div><p><strong>错误信息</strong></p>     <p>Over max limits.</p>     <p><strong>错误描述</strong></p>     <p>超过数据库订阅数目或者打开结果集数目最大支持上限。</p>     <p><strong>可能原因</strong></p>     <ol>      <li>在调用订阅数据库变化接口on时，对数据库的订阅数量已超过最大限制数目8。</li>      <li>调用获取数据库结果集接口getResultSet时，数据库当前打开的结果集数目超过最大限制数目8。</li>     </ol>     <p><strong>处理步骤</strong></p>     <ol>      <li>如果在调用订阅数据库变化接口on时，对数据库的订阅数量已超过最大限制，取消对数据库的部分订阅后，再次尝试订阅。</li>      <li>如果在调用获取数据库结果集接口getResultSet时数据库当前打开的结果集数目超过最大限制，关闭部分打开的结果集后重试。</li>     </ol> <p><strong>错误描述</strong></p>     <p>该错误码表示在调用getKVStore接口打开已创建的数据库时，options配置参数发生变化。</p>     <p><strong>可能原因</strong></p>     <p>打开已创建的数据库时，options参数配置发生了变化，可能原因如下：</p>     <ol>      <li>期望新建数据库时，使用了已创建过的数据库名称storeId。</li>      <li>期望改变已创建数据库的options参数配置。</li>     </ol>     <p><strong>处理步骤</strong></p>     <ol>      <li>新建数据库前，请检查数据库名称storeId不与已创建数据库的storeId重名。</li>      <li>期望改变已创建数据库的options参数配置时，当前不支持该操作，请自行删除数据库后使用新的options参数重新创建。</li>     </ol>    </div>    <div class="tiledSection">     <h2 id="section15100003-数据库损坏" <p><strong>可能原因</strong></p>     <p>调用数据库增、删、查、数据同步等接口操作数据库时，数据库文件已损坏。</p>     <p><strong>处理步骤</strong></p>     <ol>      <li>如果之前备份过数据库，可尝试使用已备份的数据库文件恢复数据库。</li>
