<h1 _ngcontent-vlx-c119="" class="doc-title ng-star-inserted" title="显式动画立即下发 (animateToImmediately)"> 显式动画立即下发 (animateToImmediately) </h1>

<div _ngcontent-vlx-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;"> <div><p>animateToImmediately接口用来提供<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-explicit-animation">显式动画</a>立即下发功能。同时加载多个属性动画的情况下，使用该接口可以立即执行闭包代码中状态变化导致的过渡动效。</p> <p>与<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-uicontext#animateto">animateTo</a>相比，animateToImmediately能即时将生成的动画指令发送至渲染层执行，无需等待vsync信号，从而在视觉效果上实现部分动画的优先呈现。当应用的主线程存在耗时操作，且需提前更新部分用户界面时，此接口可有效缩短应用的响应延迟。值得注意的是，animateToImmediately仅支持渲染层上的属性动画提前执行，而无法使UI侧的逐帧属性动画提前。</p> <p>此外，animateToImmediately会将调用前的状态与animateToImmediately产生的动画一并发送至渲染层，这意味着渲染可能依据调用时的状态进行。因此，务必确保调用时的状态完整无缺，能够支持当前状态下的渲染，否则动画初始阶段可能因部分状态设置不当，导致前几帧显示异常。</p> <p>通常情况下，不建议开发者采用animateToImmediately接口，而应选择<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-uicontext#animateto">animateTo</a>，以防止干扰框架的显示时序，避免在动画启动时因状态设置不完整而导致的显示错误。</p> <div><div
