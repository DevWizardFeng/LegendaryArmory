<h1 _ngcontent-hdb-c119="" class="doc-title ng-star-inserted" title="组件尺寸变化事件"> 组件尺寸变化事件 </h1>

<div _ngcontent-hdb-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <p>该事件指组件显示的尺寸发生变化时触发的事件。</p>    <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">      <p>从API version 12开始支持。后续版本如有新增内容，则采用上角标单独标记该内容的起始版本。</p>      <p>该事件返回的宽高是组件绘制出来的宽高，可能与组件设置的宽高不同。</p>     </div></div></div>    <div class="tiledSection">     <h2 id="onsizechange" device-type="phone,2in1,tablet,wearable,tv">onSizeChange<i class="anchor-icon anchor-icon-device active" active-tips="隐藏支持设备" tips="显示支持设备"></i><i class="support-device-item">Wearable</span></div><p>onSizeChange(event: SizeChangeCallback): T</p>     <p>组件区域变化时触发该回调。仅会响应由布局变化所导致的组件尺寸发生变化时的回调。</p>     <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">       <ol>        <li>         <p>该接口在布局发生变化时触发，由于计算精度的关系，其返回值可能与真实物理尺寸存在细微的差异。</p></li>        <li>         <p>onSizeChange是布局过程中触发的同步回调，直接在其中更改状态变量存在被纳入动画闭包的风险。具体而言，动画会对比动画前的布局与动画闭包后的布局，若onSizeChange的回调在动画前的布局中同步触发，那么onSizeChange回调中所做的变更将与动画闭包中的变更一同纳入动画过程。为了避免此类问题，可在onSizeChange中使用延迟时间为0的<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-timer#settimeout">setTimeOut</a>或<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-uicontext#postframecallback12">postFrameCallback</a>，将UI处理逻辑延后至异步执行。</p></li>       </ol>
