<h1 _ngcontent-iyn-c119="" class="doc-title ng-star-inserted" title="触摸测试控制"> 触摸测试控制 </h1>

<div _ngcontent-iyn-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <p>设置组件的<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-interaction-basic-principles#触摸测试" target="_blank">触摸测试</a>类型。在ArkUI开发框架中，处理触屏事件和鼠标事件时，会在事件触发前进行按压点与组件响应热区的触摸测试，以收集需响应事件的组件。基于测试结果，框架会分发相应的事件。hitTestBehavior属性用于设置不同的触摸测试响应模式，影响触摸测试收集结果及后续事件分发。具体影响参考<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-appendix-enums#hittestmode9">HitTestMode</a>枚举说明。影响<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-events-click">点击事件</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-events-touch">触摸事件</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-events-drag-drop">拖拽事件</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-mouse-key">鼠标事件</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-events-axis">轴事件</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-events-hover">悬浮事件</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-accessibility-hover-event">无障碍悬浮事件</a>和<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-gesture-settings">手势事件</a>的分发。</p>    <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">      <ul>       <li>从API version 9开始支持。后续版本如有新增内容，则采用上角标单独标记该内容的起始版本。</li>       <li>当Stack组件中有多个节点触摸区域重叠时，如两个节点，默认只会对显示在最上层的节点做触摸测试，若需要显示在下层的节点触发触摸测试，请给显示在上层的节点设置hitTestBehavior为HitTestMode.Transparent。</li>      </ul>     </div></div></div>    <div class="tiledSection">     <h2
