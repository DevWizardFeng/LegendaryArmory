<h1 _ngcontent-bwh-c119="" class="doc-title ng-star-inserted" title="Code Linter代码检查"> Code Linter代码检查 </h1>

<div _ngcontent-bwh-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div id="body0000001363071681">    <p id="ZH-CN_TOPIC_0000002478174540__p1114913186423">Code Linter支持对模块内文件或文件夹中的代码进行最佳实践/编程规范方面的检查。检查规则支持配置，配置方式请参考<a href="/consumer/cn/doc/harmonyos-guides/ide-code-linter#section19310459444">配置代码检查规则</a>。</p>    <p id="ZH-CN_TOPIC_0000002478174540__zh-cn_topic_0000001266884668_p57403171610">开发者可根据扫描结果中告警提示手工修复代码缺陷，或者执行一键式自动修复，在代码开发阶段，确保代码质量。</p>    <div class="tiledSection">     <h2 id="section19310459444">配置代码检查规则<i class="anchor-icon anchor-icon-link" anchorid="section19310459444" tips="复制节点链接"></i></h2>          <p id="ZH-CN_TOPIC_0000002478174540__p189314457446">新建工程时，工程根目录下默认创建code-linter.json5配置文件，可对代码检查的范围及对应生效的检查规则进行配置。若使用历史工程进行开发，可在工程中右键选择<strong>Code Linter &gt; Generate Config File</strong>创建code-linter.json5配置文件。</p>     <p id="ZH-CN_TOPIC_0000002478174540__p1993945144414">其中files和ignore配置项共同确定了代码检查范围，ruleSet和rules配置项共同确定了生效的规则范围。具体配置项功能如下：</p>     <p id="ZH-CN_TOPIC_0000002478174540__p149314516441"><strong>files</strong>：配置待检查的文件名单，如未指定目录，将检查当前被选中的文件或文件夹中所有的.ets文件。</p>     <p id="ZH-CN_TOPIC_0000002478174540__p59384516442"><strong>ignore</strong>：配置无需检查的文件目录，其指定的目录或文件需使用相对路径格式，相对于code-linter.json5所在工程根目录，例如：build/**/*。</p>     <p id="ZH-CN_TOPIC_0000002478174540__p119314453448"><strong>ruleSet</strong>：配置检查使用的规则集，规则集支持一次导入多条规则。规则详情请参见<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-codelinter-rule">Code Linter代码检查规则</a>。目前支持的规则集包括：</p>     <ul id="ZH-CN_TOPIC_0000002478174540__ul11938453442">      <li id="li9933459445">通用规则@typescript-eslint</li>      <li id="li49384516443">安全规则@security</li>      <li id="li129364514447">性能规则@performance</li>      <li id="li693164518445">预览规则@previewer</li>      <li id="li89317456440">一次开发多端部署规则@cross-device-app-dev</li>      <li id="li1493154513446">ArkTS代码风格规则@hw-stylistic</li>      <li id="li2094104544410">正确性规则@correctness</li>      <li id="li74963115256">兼容性规则@compatibility       <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">         <ul id="ZH-CN_TOPIC_0000002478174540__ul1049193118253">          <li id="li1849831112510">以上规则集均分为all和recommended两种规则集。all规则集是规则全集，包含所有规则；recommended规则集是推荐使用的规则集合。all规则集包含recommended规则集。</li>          <li id="li94943192518">不在工程根目录新建code-linter.json5文件的情况下，Code Linter默认会检查@performance/recommended和@typescript-eslint/recommended规则集包含的规则。</li>         </ul>        </div></div></div></li>     </ul>     <p id="ZH-CN_TOPIC_0000002478174540__p694145204415"><strong>rules</strong>：可以基于ruleSet配置的规则集，新增额外规则项，或修改ruleSet中规则默认配置，例如：将规则集中某条规则告警级别由warn改为error。</p>     <p id="ZH-CN_TOPIC_0000002478174540__p1194174534410"><strong>overrides</strong>：针对工程根目录下部分特定目录或文件，可配置定制化检查的规则。</p>     <p id="ZH-CN_TOPIC_0000002478174540__p1394114516447"><strong>extRuleSet</strong>：配置需要检查的自定义规则，具体请参考：<a href="https://gitcode.com/openharmony-sig/homecheck/blob/master/document/developer/ExtRule自定义规则开发指南.md" target="_blank">自定义规则开发指南</a>。该字段从DevEco Studio 5.1.0 Release版本开始支持。</p>     <div
