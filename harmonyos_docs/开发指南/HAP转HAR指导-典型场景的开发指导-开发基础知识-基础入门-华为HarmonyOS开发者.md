<h1 _ngcontent-puh-c119="" class="doc-title ng-star-inserted" title="HAP转HAR指导"> HAP转HAR指导 </h1>

<div _ngcontent-puh-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <p>HAP不支持导出接口或ArkUI组件给其他模块或应用使用，如果需要导出模块中的接口或ArkUI组件，并将模块作为二方库、三方库共享给其他模块或应用，可以使用HAR。本文介绍如何通过配置项的变更将HAP工程变成HAR工程。</p>    <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">      <p>部分组件和模块在HAP、HSP、HAR中集成使用时存在差异，例如<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/worker-introduction#文件路径注意事项">加载HAR中Worker线程文件相比HSP存在单独的使用约束</a>，因此按照如下步骤完成HAP转HAR后，请关注对应组件和模块介绍并进行适配。</p>     </div></div></div>    <div class="tiledSection">     <h2 id="hap转har的操作步骤">HAP转HAR的操作步骤<i class="anchor-icon anchor-icon-link" anchorid="hap转har的操作步骤" tips="复制节点链接"></i></h2>          <ol>      <li>       <p>修改HAP模块下的<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/module-configuration-file">module.json5</a>文件，具体操作如下：</p>       <ul>        <li>将type标签值改为har，删除mainElement、deliveryWithInstall、installationFree和pages标签。</li>        <li>由于API version 17及之前版本HAR不支持创建任何ExtensionAbility，从API version 18开始HAR仅支持创建<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-add-new-ability#section18891639459" target="_blank">两种ExtensionAbility</a>，因此在API version 18及之后版本但未配置为支持的两种ExtensionAbility、或在API version 17及之前版本，需要删除extensionAbilities标签，并将关联的ExtensionAbility组件删除或迁移到其他HAP模块中。</li>        <li>由于HAR模块在API version 13及以下不支持UIAbility，因此在API version 13及以前的版本，需要删除abilities标签，并将关联的UIAbility组件删除或迁移到其他HAP模块中。</li>       </ul>       <div _ngcontent-puh-c106="" class="highlight-div"><div _ngcontent-puh-c106="" class="highlight-div-header"><div
