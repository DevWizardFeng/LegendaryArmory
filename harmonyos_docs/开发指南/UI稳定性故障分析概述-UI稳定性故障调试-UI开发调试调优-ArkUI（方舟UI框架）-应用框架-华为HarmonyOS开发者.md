<p>本文档将简单介绍稳定性故障的概念与分类，并提供各类稳定性问题的参考帮助，用于指导应用开发者充分利用系统提供的调试能力和工具定位各类稳定性问题。</p>    <p>应用稳定性问题主要包括应用崩溃、应用无响应、应用资源泄漏等问题。性能问题不属于稳定性范畴，要了解性能相关内容可以参考<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ui-performance-overview">UI性能优化概览</a>。</p>    <div class="tiledSection">     <h2 id="应用崩溃">应用崩溃<i class="anchor-icon anchor-icon-link" anchorid="应用崩溃" tips="复制节点链接"></i></h2>          <p>应用崩溃有时也被称为应用闪退，指应用在运行过程中意外终止。应用异常退出的主要原因可以归纳为：</p>     <ul>      <li>遇到未处理的JS异常，如TypeError、ReferenceError等。</li>      <li>遇到未处理的native异常，如SIGSEGV、SIGABRT等。</li>      <li>遇到系统资源限制被系统终止，如内存不足、文件句柄不足、线程句柄不足等。</li>      <li>应用UI无响应被系统终止。</li>     </ul>     <p>本文档中提及的应用崩溃问题，默认指的是前两类，分别称为jscrash问题和cppcrash问题。而其他类型的崩溃问题，则分别称为应用资源泄漏问题和应用无响应问题。</p>    </div>    <div class="tiledSection">     <h3 id="jscrash" class="firsth2">jscrash<i class="anchor-icon
