<h1 _ngcontent-nvn-c119="" class="doc-title ng-star-inserted" title="使用组件截图（ComponentSnapshot）"> 使用组件截图（ComponentSnapshot） </h1>

<div _ngcontent-nvn-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <div class="tiledSection">     <h2 id="能力介绍">能力介绍<i class="anchor-icon anchor-icon-link" anchorid="能力介绍" tips="复制节点链接"></i></h2>          <p>组件截图是将应用内一个组件节点树的渲染结果生成位图（<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-image-pixelmap" target="_blank">PixelMap</a>）的能力，支持两种方式：一种是对已挂树显示的组件进行截图，另一种是对通过Builder或ComponentContent实现的离线组件进行截图。</p>     <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">       <p>组件截图依赖UI上下文，需要在具备明确上下文的环境中调用，因此请优先使用UIContext的getComponentSnapshot接口返回的<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-componentsnapshot" target="_blank">ComponentSnapshot</a>对象的接口，不建议直接使用从@kit.ArkUI导入的componentSnapshot接口。</p>      </div></div></div>    </div>    <div class="tiledSection">     <h3 id="对挂树组件截图" class="firsth2">对挂树组件截图<i class="anchor-icon anchor-icon-link" anchorid="对挂树组件截图" tips="复制节点链接"></i></h3>          <p>对已明确挂树的组件进行截图，可通过<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-componentsnapshot#get12-1" target="_blank">get</a>或<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-componentsnapshot#getsync12" target="_blank">getSync</a>实现，传入组件标识（需提前通过.id通用属性配置）以指定组件根节点。系统在通过指定的ID查找待截图组件时，仅遍历已挂树的组件，不对cache或离屏组件进行查找。系统以首个查找到的结果为准，故应用需<strong>确保组件标识ID的唯一性</strong>。</p>     <p>从API version 15开始，在已知组件的<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-arkui-framenode#getuniqueid12" target="_blank">getUniqueId</a>的情况下，也可以使用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-componentsnapshot#getwithuniqueid15" target="_blank">getWithUniqueId</a>或<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-componentsnapshot#getsyncwithuniqueid15" target="_blank">getSyncWithUniqueId</a>接口来实现截图，这可以省去查找组件的过程。</p>     <p>截图仅能获取最近一帧的绘制内容。若在组件触发更新的同时调用截图，更新的渲染内容不会被截取，截图将返回前一帧的绘制内容。</p>     <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">       <p>尽量避免在使用截图时触发待截图组件的刷新，防止对截图内容的干扰。</p>      </div></div></div>    </div>    <div class="tiledSection">     <h3 id="对离线组件截图">对离线组件截图<i class="anchor-icon anchor-icon-link" anchorid="对离线组件截图" tips="复制节点链接"></i></h3>          <p>离线组件是指通过Builder或ComponentContent封装的、尚未挂载到树上的组件，可以使用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-componentsnapshot#createfrombuilder12-1" target="_blank">createFromBuilder</a>来实现。从API version 18开始，也可以使用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-componentsnapshot#createfromcomponent18" target="_blank">createFromComponent</a>实现离线组件截图。</p>     <p>这些组件不参与真实渲染，因此对其截图需要更长的时间，因为系统必须先进行离线构建、布局及资源加载等操作，在这些操作完成前执行的截图所获位图不符合预期。因此，通常需要通过设置delay参数指定足够的时间，确保系统能够完成这些操作。对于图片资源的加载，建议将图片组件的<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-image#syncload8" target="_blank">syncLoad</a>属性设为 true，以强制同步加载，确保离线组件构建时图片已加载、下载及解码完成，从而确保截图过程中能够正确呈现图片像素。</p>    </div>    <div class="tiledSection">     <h2 id="典型使用场景">典型使用场景<i class="anchor-icon anchor-icon-link" anchorid="典型使用场景" tips="复制节点链接"></i></h2>          <p>以下通过几个典型场景来说明组件截图能力的常见使用方式。</p>    </div>    <div class="tiledSection">     <h3 id="截取长内容滚动截图" class="firsth2">截取长内容（滚动截图）<i class="anchor-icon anchor-icon-link" anchorid="截取长内容滚动截图" tips="复制节点链接"></i></h3>          <p>较长内容通常使用滚动类容器组件实现。截图时，仅能捕获容器内可见内容，超出边界部分无法截取。若使用LazyForEach或Repeat，超出显示范围内容亦不会被系统构建及截取。</p> id="组件截图最佳实践">组件截图最佳实践<i class="anchor-icon anchor-icon-link" anchorid="组件截图最佳实践" tips="复制节点链接"></i></h2>         </div>    <div class="tiledSection">     <h3 id="合理控制截图时机" class="firsth2">合理控制截图时机<i class="anchor-icon anchor-icon-link" anchorid="合理控制截图时机" tips="复制节点链接"></i></h3>          <p>在实现截图功能时，需注意组件的渲染过程非一次性完成。系统在构建与显示组件时，将经过测量、布局、提交指令等多个复杂步骤，最终在一次硬件刷新时呈现于屏幕上。因此，在特定情况下，若在组件刷新后立即调用截图，可能无法获取预期内容或出现截图失败报错。</p>     <p>为了确保截图结果准确，建议在组件完全渲染后再执行截图操作。</p>     <p><strong>了解组件的绘制状态</strong></p>     <p>为了确保截图内容符合预期，应该了解代码对界面状态的修改时机，并注意给系统预留处理时间，这通常可以通过增加一定延时来实现。</p>     <p>尽管可以通过inspector上的<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-arkui-inspector#componentobserver" target="_blank">ComponentObserver</a>感知应用组件绘制（draw）送显通知，但需要注意的是，ComponentObserver的组件绘制通知并不意味着系统已经真正将绘制指令执行，这取决于图形系统服务的负载情况。</p>     <p><strong>明确等待绘制完成</strong></p>     <p>影响截图预期的主要因素是截图时机与系统服务执行绘制指令的时间差。在发起截图调用时，应用侧之前提交的所有绘制指令可能尚未被图形服务真正执行。为此，可以通过指定<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-arkui-componentsnapshot#snapshotoptions12" target="_blank">SnapshotOptions</a>参数中的waitUntilRenderFinished为true，来确保系统在执行截图请求时等待所有之前的绘制指令均执行完毕，从而截取到更完整的内容。</p>     <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">       <p>建议始终开启waitUntilRenderFinished参数。</p>      </div></div></div>     <p><strong>了解资源加载对截图的影响</strong></p>     <p>影响截图预期的另一个常见原因，是图片资源的加载。图片组件支持在线资源链接，也可指定本地资源，且绝大多数图片资源为PNG、JPEG等压缩格式。这些资源需要系统解码为可提交绘制的位图格式，此过程默认在异步IO线程上进行，因此可能由于该过程耗时的不确定性而导致截图不符合预期。</p>     <p>应用可通过以下几种方式进行优化：</p>     <ol>      <li>自行提前解析图片为PixelMap格式，将PixelMap配置给图片组件；建议优先以此方法进行优化。</li>      <li>配置所使用的图片组件的syncload属性为true来强制同步加载，这样组件被构建时，即可确保资源可以直接被提交；</li>      <li>通过指定延迟时长以及checkImageStatus设置为true，尝试截图，当返回160001错误后，重新加大时长进行截图；</li>     </ol>    </div>    <div class="tiledSection">     <h3 id="及时保存和释放位图对象">及时保存和释放位图对象<i class="anchor-icon anchor-icon-link" anchorid="及时保存和释放位图对象" tips="复制节点链接"></i></h3>          <p>为了及时释放资源，当截图接口返回的PixelMap对象不再使用时，应将其赋值为空。</p>    </div>    <div class="tiledSection">     <h3 id="合理控制采样精度">合理控制采样精度<i class="anchor-icon anchor-icon-link" anchorid="合理控制采样精度" tips="复制节点链接"></i></h3>          <p>请不要截取过大尺寸的图片，截图不建议超过屏幕尺寸大小。当要截取的图片目标长宽超过底层限制时，截图会返回失败，不同设备的底层限制并不相同。可以通过控制SnapshotOptions中的scale参数，减小采样精度，这可以在很大程度上节省内存，并大幅度提高截图的效率。</p>    </div>    <div class="tiledSection">     <h3 id="使用其他能力对自渲染场景实现截图">使用其他能力对自渲染场景实现截图<i class="anchor-icon anchor-icon-link" anchorid="使用其他能力对自渲染场景实现截图" tips="复制节点链接"></i></h3>          <p>尽管截图只需传入一个组件根节点即可实现对其下所有组件进行截图，但当子组件中存在<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-media-components-video" target="_blank">Video</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-xcomponent" target="_blank">XComponent</a>或<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-basic-components-web" target="_blank">Web</a>组件时，这并不是推荐的截图方式。建议直接使用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-image-f#imagecreatepixelmapfromsurface11" target="_blank">image.createPixelMapFromSurface</a>接口来实现。</p>    </div>
