<h1 _ngcontent-tuj-c119="" class="doc-title ng-star-inserted" title="分布式数据对象跨设备数据同步 (ArkTS)"> 分布式数据对象跨设备数据同步 (ArkTS) </h1>

<div _ngcontent-tuj-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <div class="tiledSection">     <h2 id="场景介绍">场景介绍<i class="anchor-icon anchor-icon-link" anchorid="场景介绍" tips="复制节点链接"></i></h2>          <p>传统方式下，设备之间的数据同步，需要开发者完成消息处理逻辑，包括：建立通信链接、消息收发处理、错误重试、数据冲突解决等操作，工作量非常大。而且设备越多，调试复杂度也将同步增加。</p>     <p>其实设备之间的状态、消息发送进度、发送的数据等都是“变量”。如果这些变量支持“全局”访问，那么开发者跨设备访问这些变量就能像操作本地变量一样，从而能够自动高效、便捷地实现数据多端同步。</p>     <p>分布式数据对象即实现了对“变量”的“全局”访问。向应用开发者提供内存对象的创建、查询、删除、修改、订阅等基本数据对象的管理能力，同时具备分布式能力。为开发者在分布式应用场景下提供简单易用的JS接口，轻松实现多设备间同应用的数据协同，同时设备间可以监听对象的状态和数据变更。满足超级终端场景下，相同应用多设备间的数据对象协同需求。与传统方式相比，分布式数据对象大大减少了开发者的工作量。</p>     <p>目前分布式数据对象只能在<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/app-continuation-guide" target="_blank">跨端迁移</a>和通过跨设备Call调用实现的多端协同场景中使用。</p>    </div>    <div class="tiledSection">     <h2 id="基本概念">基本概念<i class="anchor-icon anchor-icon-link" anchorid="基本概念" tips="复制节点链接"></i></h2>          <ul>      <li>       <p><strong>分布式内存数据库</strong>：分布式内存数据库将数据缓存在内存中，以便应用获得更快的数据存取速度，不会将数据进行持久化。若数据库关闭，则数据不会保留。</p></li>      <li>       <p><strong>分布式数据对象</strong>：分布式数据对象是一个JS对象型的封装。每一个分布式数据对象实例会创建一个内存数据库中的数据表，每个应用程序创建的内存数据库相互隔离，对分布式数据对象的“读取”或“赋值”会自动映射到对应数据库的get/put操作。</p>       <p>分布式数据对象的生命周期包括以下状态：</p>       <ul>        <li>未初始化：未实例化，或已被销毁。</li>        <li>本地数据对象：已创建对应的数据表，但是还无法进行数据同步。</li>        <li>分布式数据对象：已创建对应的数据表，设备在线且组网内设置同样sessionId的对象数&gt;=2，可以跨设备同步数据。若设备掉线或将sessionId置为空，分布式数据对象退化为本地数据对象。</li>       </ul></li>     </ul>    </div>    <div class="tiledSection">     <h2 id="运作机制">运作机制<i class="anchor-icon anchor-icon-link" anchorid="运作机制" tips="复制节点链接"></i></h2>          <p><strong>图1</strong> id="跨设备同步和数据变更通知机制">跨设备同步和数据变更通知机制<i class="anchor-icon anchor-icon-link" anchorid="跨设备同步和数据变更通知机制" tips="复制节点链接"></i></h3>          <p>分布式数据对象，最重要的功能就是对象之间的数据同步。可信组网内的设备可以在本地创建分布式数据对象，并设置sessionID。不同设备上的分布式数据对象，通过设置相同的sessionID，建立对象之间的同步关系。</p>     <p>如下图所示，设备A和设备B上的“分布式数据对象1”，其sessionID均为session1，这两个对象建立了session1的同步关系。</p>     <p><strong>图2</strong> 对象的同步关系</p>     <p><span><img originheight="249" originwidth="831" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20251211114736.70557775816681809692772899529592:50001231000000:2800:59251DDDAFD1C1E7734D99AA6F13B43D4927073DBC631C31C76F19DA40D579C6.jpg" width="831" height="249"></span></p>     <p>一个同步关系中，一个设备只能有一个对象加入。比如上图中，设备A的“分布式数据对象1”已经加入了session1的同步关系，所以设备A的“分布式数据对象2”就加入失败了。</p>     <p>建立同步关系后，每个Session有一份共享对象数据。加入了同一个Session的对象，支持以下操作：</p>     <p>（1）读取/修改Session中的数据。</p>     <p>（2）监听数据变更，感知其他设备对共享对象数据的修改。</p>     <p>（3）监听状态变更，感知其他设备的加入和退出。</p>     <p>分布式数据对象加入session时，如果它的数据与session中的数据不同，则它会更新session中的数据。如果希望分布式数据对象加入sessionId时不更新session中的数据，并且得到session中的数据，需要将对象的属性的值设置为undefined（资产类型的属性则是将它的各个属性值设置为空字符串）。</p>    </div>    <div class="tiledSection">     <h3 id="同步的最小单位">同步的最小单位<i class="anchor-icon anchor-icon-link" anchorid="同步的最小单位" tips="复制节点链接"></i></h3>          <p>关于分布式数据对象的数据同步，值得注意的是，同步的最小单位是“属性”。比如，下图中对象1包含三个属性：name、age和parents。当其中一个属性变更时，则数据同步时只需同步此变更的属性。</p>     <p>对象属性支持基本类型（数字类型、布尔类型、字符串类型）以及复杂类型（数组、基本类型嵌套）。针对复杂类型的数据修改，目前仅支持对根属性的修改，暂不支持对下级属性的修改。</p>   <p>分布式对象主要运行在应用程序的进程空间。当调用分布式对象持久化接口时，通过分布式数据库对对象进行持久化和同步，进程退出后数据也不会丢失。</p>     <p>该场景是分布式对象的扩展场景，主要用于以下情况：</p>     <ul>      <li>       <p>在设备上创建持久化对象后APP退出，重新打开APP，创建持久化对象，加入同一个Session，数据可以恢复到APP退出前的数据。</p></li>      <li>       <p>在设备A上创建持久化对象并同步后持久化到设备B后，A设备的APP退出，设备B打开APP，创建持久化对象，加入同一个Session，数据可以恢复到A设备退出前的数据。</p></li>     </ul>    </div>    <div class="tiledSection">     <h3 id="资产同步机制">资产同步机制<i class="anchor-icon anchor-icon-link" anchorid="资产同步机制" tips="复制节点链接"></i></h3>          <p>在分布式对象中，可以使用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-data-commontype#asset" target="_blank">资产类型</a>来描述本地实体资产文件，分布式对象跨设备同步时，该文件会和数据一起同步到其他设备上。</p>     <p>在API version 20之前版本，仅支持资产类型，不支持<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-data-commontype#assets" target="_blank">资产类型数组</a>。如需同步多个资产，可将每个资产作为分布式对象的一个根属性实现。</p>     <p>从API version 20开始，支持<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-data-commontype#assets" target="_blank">资产类型数组</a>的同步。</p>    </div>    <div class="tiledSection">     <h2 id="约束限制">约束限制<i class="anchor-icon anchor-icon-link" anchorid="约束限制" tips="复制节点链接"></i></h2>          <ul>      <li>       <p>目前分布式数据对象只能在<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/app-continuation-guide" target="_blank">跨端迁移</a>和通过跨设备Call调用实现的多端协同场景中使用。跨端迁移场景下，每个分布式数据对象大小不超过150KB；多端协同场景下，每个分布式数据对象大小不超过500KB。</p></li>      <li>       <p>当前跨设备接续能力支持以下两种场景的Ability跨端迁移</p>       <ul>        <li><a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/app-continuation-guide#section71450211617" target="_blank">支持同应用中不同Ability跨端迁移</a></li>        <li><a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/app-continuation-guide#section19162158174714" target="_blank">支持同应用不同BundleName的Ability跨端迁移</a></li>       </ul></li>      <li>       <p>分布式数据对象的数据同步发生在同一个应用程序下，且同sessionID之间。</p></li>      <li>       <p>设备A修改1KB数据，设备B收到变更通知，50ms内完成。</p></li>      <li>       <p>单个应用程序最多只能创建16个分布式数据对象实例。</p></li>      <li>       <p>考虑到性能和用户体验，最多不超过3个设备进行数据协同。</p></li>      <li>       <p>如对复杂类型的数据进行修改，仅支持修改根属性，暂不支持下级属性修改。<a href="/consumer/cn/doc/harmonyos-guides/data-sync-of-distributed-data-object#资产同步机制">资产同步机制</a>中，资产类型的数据支持下一级属性修改。</p></li>      <li>       <p>支持JS接口间的互通，与其他语言不互通。</p></li>     </ul>    </div>    <div class="tiledSection">     <h2 id="开发步骤">开发步骤<i class="anchor-icon anchor-icon-link" anchorid="开发步骤" tips="复制节点链接"></i></h2>         </div>    <div class="tiledSection">     <h3 id="在跨端迁移中使用分布式数据对象迁移数据" class="firsth2">在跨端迁移中使用分布式数据对象迁移数据<i class="anchor-icon anchor-icon-link" anchorid="在跨端迁移中使用分布式数据对象迁移数据" tips="复制节点链接"></i></h3>          <ol>      <li>       <p>迁移发起端在onContinue接口中创建分布式数据对象并保存数据到接收端：</p>       <p>1.1 调用create接口创建并得到一个分布式数据对象实例。</p>       <p>1.2 调用genSessionId接口创建一个sessionId，调用setSessionId接口设置同步的sessionId，并将这个sessionId放入wantParam。</p>       <p>1.3 从wantParam获取接收端设备networkId，使用这个networkId调用save接口保存数据到接收端。</p></li>      <li>       <p>接收端在onCreate和onNewWant接口中创建分布式数据对象并注册恢复状态监听：</p>       <p>2.1 调用create接口创建并得到一个分布式数据对象实例。</p>       <p>2.2 注册恢复状态监听。收到状态为'restored'的回调通知时，表示接收端分布式数据对象已恢复发起端保存过来的数据。</p>       <p>2.3 从want.parameters中获取发起端放入的sessionId，调用setSessionId接口设置同步的sessionId。</p></li>     </ol>     <div><div class="hw-editor-tip <p>在应用迁移启动时，无论是冷启动还是热启动，都会在执行完onCreate()/onNewWant()后，触发<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-app-ability-uiability#onwindowstagerestore" <p>wantParam中的"sessionId"字段可能被其他服务占用，建议自定义一个key存取sessionId。</p></li>        <li>         <p>可以使用资产类型记录资产附件（文件、图片、视频等类型文件）的相关信息，迁移资产类型数据时，对应的资产附件会一起迁移到对端。</p></li>        <li>         <p>接收端需要将业务数据的初始值设置为undefined，才能恢复发起端保存的数据，否则接收端的数据会覆盖同步到发起端。如果是资产数据，需要将资产数据的各个属性设置为空字符串而不是将整个资产数据设置为undefined。</p></li>        <li>         <p>API version
