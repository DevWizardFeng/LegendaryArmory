<h1 _ngcontent-ure-c119="" class="doc-title ng-star-inserted" title="分段加解密说明"> 分段加解密说明 </h1>

<div _ngcontent-ure-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <p>在加解密的过程中，算法库没有对单次或累计的传入数据量设置大小限制，但在传入的数据量较大时（如数据量大于2M），建议开发者将数据分段，完成分段加解密，提高效率。</p>    <div class="tiledSection">     <h2 id="对称加解密">对称加解密<i class="anchor-icon anchor-icon-link" anchorid="对称加解密" tips="复制节点链接"></i></h2>          <p>对称密钥的分段加解密，通过调用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-cryptoframework#update-1" target="_blank">Cipher.update</a>实现。</p>     <p>开发者可自定义单次传入的数据量（示例中的updateLength），多次调用update传入数据。</p>     <p>当前单次支持传入的最大长度为INT_MAX（Uint8Array类型的最大长度）。</p>     <p><strong>开发者应当对每次update和doFinal的结果都判断是否为null，并在结果不为null时取出其中的数据进行拼接，形成完整的密文/明文。</strong></p>     <p><strong>开发示例：</strong> <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/crypto-aes-sym-encrypt-decrypt-gcm-by-segment">使用AES对称密钥（GCM模式）分段加解密</a></p>     <p><strong>开发示例：</strong> <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/crypto-sm4-sym-encrypt-decrypt-gcm-by-segment">使用SM4对称密钥（GCM模式）分段加解密</a></p>    </div>    <div class="tiledSection">     <h2 id="非对称加解密">非对称加解密<i class="anchor-icon anchor-icon-link" anchorid="非对称加解密" tips="复制节点链接"></i></h2>          <p>非对称加解密，不支持update操作，仅需要调用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-cryptoframework#init-1" target="_blank">Cipher.init</a>和<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-cryptoframework#dofinal-1" target="_blank">Cipher.doFinal</a>完成加解密。</p>     <p>非对称密钥的分段加密是指当明文大于单次加密支持的数据长度时（具体长度请查<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/crypto-asym-encrypt-decrypt-spec">非对称密钥加解密算法规格</a>），需要将待加密数据分为合适长度的数据段，并对每个数据段执行加密操作，即创建Cipher，然后调用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-cryptoframework#init-1" target="_blank">Cipher.init</a>和<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-cryptoframework#dofinal-1" target="_blank">Cipher.doFinal</a>接口。</p>     <p>严格意义上说，是数据的拆分加解密，此时单次传入的数据量长度与密钥规格的长度相关。</p>     <ul>      <li>RSA：填充模式不同，输入的数据的规则不同。请根据<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/crypto-asym-encrypt-decrypt-spec#rsa">RSA算法规格</a>确认单次传入的数据量长度。</li>     </ul>     <p><strong>开发者应当对每次doFinal的结果都判断是否为null，并在结果不为null时取出其中的数据进行拼接，形成完整的密文/明文。</strong></p>     <p><strong>开发示例：</strong> <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/crypto-rsa-asym-encrypt-decrypt-by-segment">使用RSA非对称密钥分段加解密</a></p>    </div>    <div class="tiledSection">     <h2 id="常见问题">常见问题<i class="anchor-icon anchor-icon-link" anchorid="常见问题" tips="复制节点链接"></i></h2>          <ul>      <li>       <p>分段加解密中，每次更新的数据量，与加密模式是否有关？</p>       <p>每次更新的数据量，由开发者自定义，与加密模式无关。</p>       <p>不同的加密模式，仅对加解密参数产生影响，不同的加密模式使用的加解密参数不同，具体请参考<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-cryptoframework#paramsspec" target="_blank">ParamsSpec</a>。</p></li>     </ul>    </div>   </div>   <div></div></div>
