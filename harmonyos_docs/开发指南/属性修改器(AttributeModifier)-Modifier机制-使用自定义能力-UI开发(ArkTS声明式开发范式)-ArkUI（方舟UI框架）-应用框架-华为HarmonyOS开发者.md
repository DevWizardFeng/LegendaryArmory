class="anchor-icon anchor-icon-link" anchorid="概述" tips="复制节点链接"></i></h2>          <p>声明式语法引入了<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-style">@Styles</a>和<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-extend">@Extend</a>两个装饰器，可以解决复用相同自定义样式的问题，但是存在以下受限场景：</p>     <ul>      <li>@Styles和@Extend均是编译期处理，不支持跨文件的导出复用。</li>      <li>@Styles仅能支持通用属性、事件，不支持组件特有的属性。</li>      <li>@Styles虽然支持在多态样式下使用，但不支持传参，无法对外开放一些属性。</li>      <li>@Extend虽然能支持特定组件的私有属性、事件，但同样不支持跨文件导出复用。</li>      <li>@Styles、@Extend对于属性设置，无法支持业务逻辑编写，动态决定是否设置某些属性，只能通过三元表达式对所有可能设置的属性进行全量设置，设置大量属性时效率较低。</li>     </ul>     <p>为了解决上述问题，ArkUI引入了AttributeModifier机制，可以通过Modifier对象动态修改属性。能力对比如下：</p>     <div class="tiledSection">     <h2 id="使用说明">使用说明<i class="anchor-icon anchor-icon-link" anchorid="使用说明" tips="复制节点链接"></i></h2>          <ul>      <li>组件通用方法attributeModifier支持传入一个实现AttributeModifier&lt;T&gt;接口的实例，T必须指定为组件对应的Attribute类型，或者是CommonAttribute。</li>      <li>在组件首次初始化或者关联的状态变量发生变化时，如果传入的实例实现了对应接口，会触发applyNormalAttribute。</li>      <li>回调applyNormalAttribute时，会传入组件属性对象，通过该对象可以设置当前组件的属性/事件。</li>      <li>暂未支持的属性/事件，执行时会抛异常。</li>      <li>属性变化触发applyXxxAttribute函数时，该组件之前已设置的属性，在本次变化后未设置的属性会恢复为属性的默认值。</li>      <li>可以通过该接口使用多态样式的功能，例如如果需要在组件进入按压态时设置某些属性，就可以通过自定义实现applyPressedAttribute方法完成。</li>      <li>一个组件上同时使用属性方法和applyNormalAttribute设置相同的属性，遵循属性覆盖原则，即后设置的属性生效。</li>      <li>一个Modifier实例对象可以在多个组件上使用。</li>      <li>一个组件上多次使用applyNormalAttribute设置不同的Modifier实例，每次状态变量刷新均会按顺序执行这些实例的方法属性设置，同样遵循属性覆盖原则。</li>     </ul>    </div>    <div class="tiledSection">     <h2 id="设置和修改组件属性">设置和修改组件属性<i
