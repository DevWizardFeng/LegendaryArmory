<div _ngcontent-whg-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;"> <div><p>设备的基准宽度为720px（px为逻辑像素，非物理像素），实际显示效果会根据实际屏幕宽度进行缩放。</p> <p>其换算关系如下：</p> <p>组件的width设为100px时，在宽度为720物理像素的屏幕上，实际显示为100物理像素；在宽度为1440物理像素的屏幕上，实际显示为200物理像素。</p> <p>一个页面的基本元素包含标题区域、文本区域、图片区域等，每个基本元素内还可以包含多个子元素，开发者根据需求还可以添加按钮、开关、进度条等组件。在构建页面布局时，需要对每个基本元素思考以下几个问题：</p> <ul><li><p>该元素的尺寸和排列位置</p>  </li><li><p>是否有重叠的元素</p>  </li><li><p>是否需要设置对齐、内间距或者边界</p>  </li><li><p>是否包含子元素及其排列位置</p>  </li><li><p>是否需要容器组件及其类型</p>  </li></ul> <p>将页面中的元素分解之后再对每个基本元素按顺序实现，可以减少多层嵌套造成的视觉混乱和逻辑混乱，提高代码的可读性，方便对页面做后续的调整。以下图为例进行分解：</p> <p><strong>图1</strong> 页面布局分解</p> <p><span><img originheight="897" originwidth="872" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20251211114904.89236943229255043715107574436366:50001231000000:2800:9800B3E40F48438E5826F6E9EAA3F65DFF0E42B31D5C4D1374CBF3C8532998D3.png" width="872" height="897"></span></p> <p><strong>图2</strong> 留言区布局分解</p> <p><span><img originheight="289" originwidth="648" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20251211114904.74507247079054899793627138548120:50001231000000:2800:D1B17C064B4EC7E5680E2B727BD836B9BA4A33F40195BC3C3CA0ADF5C0D47A55.png" width="648" height="289"></span></p> </div> <div></div></div>
