anchor-icon-link" anchorid="应用模型" tips="复制节点链接"></i></h3><p>应用模型是HarmonyOS为开发者提供的应用程序所需能力的抽象提炼，它提供了应用程序必备的组件和运行机制。有了应用模型，开发者可以基于一套统一的模型进行应用开发，使应用开发更简单、高效。</p> <p>随着系统的演进发展，HarmonyOS先后提供了两种应用模型：</p> <ul><li><strong>FA（Feature Ability）模型：</strong> HarmonyOS API 7开始支持的模型，已经不再主推。FA模型开发可见<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/fa-model-development-overview" target="_blank">FA模型开发概述</a>。<strong>快速入门章节不再对此展开提供开发指导。</strong></li><li><p><strong>Stage模型：</strong> HarmonyOS API 9开始新增的模型，是目前主推且会长期演进的模型。在该模型中，由于提供了AbilityStage、WindowStage等类作为应用组件和Window窗口的“舞台”，因此称这种应用模型为Stage模型。Stage模型开发可见<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/stage-model-development-overview" target="_blank">Stage模型开发概述</a>。<strong>快速入门以此为例提供开发指导。</strong></p> </li></ul> <p>FA模型和Stage模型的整体架构和设计思想等更多区别，请见<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/application-models" target="_blank">应用模型</a>。</p> <p>快速入门提供了一个含有两个页面的开发实例，并基于Stage模型构建第一个ArkTS应用，以便开发者理解以上基本概念及应用开发流程。</p> </div> <div class="tiledSection"><h2 id="工具准备">工具准备<i class="anchor-icon anchor-icon-link" anchorid="工具准备"
