<h1 _ngcontent-pwj-c119="" class="doc-title ng-star-inserted" title="弹窗概述"> 弹窗概述 </h1>

<div _ngcontent-pwj-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <p>弹窗一般指打开应用时自动弹出或者用户行为操作时弹出的UI界面，用于短时间内展示用户需关注的信息或待处理的操作。</p>    <p>从ArkUI组件树层级上来看，Overlay浮层、弹窗、模态、带Order的Overlay浮层都挂载在Root节点下。弹窗、模态、带Order的Overlay浮层根据设置的层级（数字大小）进行逐层显示，并且全部显示在Overlay浮层上面。如图所示：</p>    <p><span><img originheight="597" originwidth="1508" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20251211114754.18135905555952789458807898875148:50001231000000:2800:D763F7096C6AD7AD562A2B4D1E802141EB6F67C4FAE6F2776AE53FB72156FC22.png" width="920" height="364.2175066312998"></span></p>    <p>对于一个多页面应用，基本树结构如下所示。多个Page页面之间使用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-router" target="_blank">Router</a>的接口进行跳转。<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-navigation#navbar12" target="_blank">NavBar</a>和<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-navdestination" target="_blank">Navdestination</a>组成的页面可以通过<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-navigation" target="_blank">Navigation</a>接口进行跳转。</p>    <p><span><img originheight="602" originwidth="600" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20251211114754.20379508071634879020436860176668:50001231000000:2800:3097F58F81FD6F63CCD46D3897E75CD2937D1E84949983225ABDEAFD606D4A20.png" width="600" height="602"></span></p>    <p>而<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-base-dialog-overview">Dialog</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-popup-overview">Popup</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-menu-overview">Menu</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-create-overlaymanager">OverlayManager</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-create-toast">Toast</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-sheet-page">bindSheet</a>以及<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-contentcover-page">bindContentCover</a>的组件在结合路由、导航使用时存在如下几种显示模式。</p>    <ul>     <li>      <p>弹窗显示在当前应用窗口最上层，层级高于应用主窗内所有页面（默认行为）。</p>      <p>如下图所示，触发弹窗显示时，弹窗层级要高于Page页面和Navigation页面，即如果弹窗组件没有被关闭的话，页面切换前后，弹窗组件都会显示在页面上方，而不会出现新的路由/导航页面覆盖在弹窗之上的情况。</p>      <p><span><img originheight="437" originwidth="800" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20251211114754.61949860707643399692744928782313:50001231000000:2800:91BCF76DBBD67C9D18F1FFC77BEFC495068C65CCA48EF0E3982CD5A548A39D7B.png" width="800" height="437"></span></p>      <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">        <p>如果Popup或Menu等存在绑定组件的弹窗组件，在页面跳转后因为所绑定的组件不在新页面显示，系统内部会自动关闭对应弹窗组件。但因为弹窗组件显示同时受开发者配置的参数控制，例如当<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-popup-overview">Popup</a>的show参数设置为显示时，弹窗组件会覆盖在下一个页面上显示。</p>       </div></div></div></li>     <li>      <p>弹窗显示在当前页面内，层级低于下一跳页面。</p>      <p>当开发者期望弹窗是一个页面内的弹窗，即当路由/导航切换页面时，弹窗会被跳转的页面覆盖，随着页面切回原页面，弹窗仍然正常显示。此时弹窗层级如下图所示：</p>      <p><span><img originheight="501" originwidth="700" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20251211114754.29150940460423472718463089641379:50001231000000:2800:7F707E472DD9D537355AEFCDCEDD84D70EC6F851E378B12A46B54E90057208E3.png" width="700" height="501"></span></p>      <p>页面级弹窗功能需要结合弹窗组件的页面级能力使用，当前支持页面级能力的有<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-base-dialog-overview">Dialog</a>和<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-sheet-page">bindSheet</a>。</p></li>     <li>      <p>弹窗显示在独立窗口内，窗口层级高于应用所在窗口。</p>      <p>在PC/2in1等设备上，开发者常期望弹窗的显示在应用窗口之外，此时需要借助子窗的能力。例如自定义弹窗可以通过<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-methods-custom-dialog-box#customdialogcontrolleroptions对象说明" target="_blank">CustomDialogControllerOptions</a>中的showInSubwindow属性控制是否使用子窗功能。基于子窗显示的弹窗效果如下图所示。</p>      <p><span><img originheight="501" originwidth="800" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20251211114754.40981462654365363233314122748264:50001231000000:2800:A03963701BD0516D3DAADE54639984B53B0BFD447176805B31A94242783FE1E6.png" width="800" height="501"></span></p>      <p>此时弹窗组件的显示层级受窗口的层级管理控制，会高于当前应用所在的窗口，低于系统窗口（如系统输入法、系统弹窗等）。</p></li>    </ul>    <div class="tiledSection">     <h2 id="弹窗的种类">弹窗的种类<i class="anchor-icon anchor-icon-link" anchorid="弹窗的种类" tips="复制节点链接"></i></h2>          <p>根据用户交互操作场景，弹窗可分为<strong>模态弹窗</strong>和<strong>非模态弹窗</strong>两种类型，其区别在于用户是否必须对其做出响应。</p>     <ul>      <li><strong>模态弹窗：</strong> 为强交互形式，会中断用户当前的操作流程，要求用户必须做出响应才能继续其他操作，通常用于需要向用户传达重要信息的场景。</li>      <li><strong>非模态弹窗：</strong> 为弱交互形式，不会影响用户当前操作行为，用户可以不对其进行回应，通常都有时间限制，出现一段时间后会自动消失。一般用于告诉用户信息内容外还需要用户进行功能操作的场景。</li>     </ul>     <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">       <p>当前模态弹窗通过设置指定属性变成非模态，例如AlertDialog，可以设置<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-methods-alert-dialog-box#alertdialogparam对象说明" target="_blank">isModal</a>为false变成非模态，其他弹窗详见API说明。</p>       <p>不建议在非前台状态下，调用弹窗显示接口。</p>       <p>系统弹窗由系统弹出，出于安全考虑，不支持自定义样式。</p>       <p>系统弹窗出现时，调用非系统弹窗的显示接口（例如：promptAction的openCustomDialog、CustomDialogController的open等），禁止该类弹窗显示。</p>      </div></div></div>
