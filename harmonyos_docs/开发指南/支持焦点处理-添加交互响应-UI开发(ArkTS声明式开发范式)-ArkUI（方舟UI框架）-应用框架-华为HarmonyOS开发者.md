<div _ngcontent-abf-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <div class="tiledSection">     <h2 id="基础概念与规范">基础概念与规范<i class="anchor-icon anchor-icon-link" anchorid="基础概念与规范" tips="复制节点链接"></i></h2>         </div>    <div class="tiledSection">     <h3 id="基础概念" class="firsth2">基础概念<i class="anchor-icon anchor-icon-link" anchorid="基础概念" tips="复制节点链接"></i></h3>          <p><strong>焦点、焦点链和走焦</strong></p>     <ul>      <li>焦点：指向当前应用界面上唯一的一个可交互元素，当用户使用键盘、电视遥控器、车机摇杆/旋钮等非指向性输入设备与应用程序进行间接交互时，基于焦点的导航和交互是重要的输入手段。</li>      <li>焦点链：在应用的组件树形结构中，当一个组件获得焦点时，从根节点到该组件节点的整条路径上的所有节点都会处于焦点状态，形成一条连续的焦点链。</li>      <li>走焦：指焦点在应用内的组件之间转移的行为。这一过程对用户是透明的，但开发者可以通过监听onFocus（焦点获取）和onBlur（焦点失去）事件来捕捉这些变化。关于走焦的具体方式和规则，详见<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#走焦规范">走焦规范</a>。</li>     </ul>     <p><strong>焦点激活态</strong></p>     <p>焦点激活态是用来显示当前获焦组件焦点框的视觉样式。</p>     <ul>      <li>       <p>显示规则</p>       <ul>        <li>默认状态：焦点激活态默认是隐藏的。</li>        <li>激活条件：只有当应用程序进入"激活态"时，焦点激活态才会显示。</li>        <li>重要关系：         <ul>          <li>获得焦点的组件不一定显示激活态（取决于应用是否处于激活态）。</li>          <li>显示激活态的组件必定是当前获得焦点的组件。</li>         </ul></li>        <li>样式定制：组件通常有内置的激活态样式，开发者可以通过样式接口自定义，自定义后会覆盖默认样式。</li>        <li>显示优先级：当多个组件同时拥有焦点时，系统优先显示子组件的激活态，且同一时间只显示一个激活态。</li>       </ul></li>      <li>       <p>如何进入激活态</p>       <ul>        <li>按下外接键盘的Tab键（注意：首次激活时的Tab键仅用于激活，不会触发焦点移动）。</li>        <li>调用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-focuscontroller" target="_blank">FocusController</a>的activate(true)方法。</li>       </ul></li>      <li>       <p>如何退出激活态</p>       <ul>        <li>调用FocusController的activate(false)方法。</li>        <li>发生点击事件时（包括触屏点击或鼠标左键点击）。</li>       </ul></li>     </ul>     <div _ngcontent-abf-c106="" class="highlight-div"><div _ngcontent-abf-c106="" class="highlight-div-header"><div _ngcontent-abf-c106="" </ol>     <p><strong>层级页面</strong></p>     <p>层级页面是焦点框架中特定容器组件的统称，涵盖Page、Dialog、SheetPage、ModalPage、Menu、Popup、NavBar、NavDestination等。这些组件通常具有以下关键特性：</p>     <ul>      <li>视觉层级独立性：从视觉呈现上看，这些组件独立于其他页面内容，并通常位于其上方，形成视觉上的层级差异。</li>      <li>焦点跟随：此类组件在首次创建并展示之后，会立即将应用内焦点抢占。</li>      <li>走焦范围限制：当焦点位于这些组件内部时，用户无法通过键盘按键将焦点转移到组件外部的其他元素上，焦点移动仅限于组件内部。</li>     </ul>     <p>在一个应用程序中，任何时候都至少存在一个层级页面组件，并且该组件会持有当前焦点。当该层级页面关闭或不再可见时，焦点会自动转移到下一个可用的层级页面组件上，确保用户交互的连贯性和一致性。</p>     <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">       <p>Popup组件在focusable属性（组件属性，非通用属性）为false的时候，不会有第2条特性。</p>       <p>NavBar、NavDestination没有第3条特性，对于它们的走焦范围，是与它们的首个父层级页面相同的。</p>      </div></div></div>     <p><strong>根容器</strong></p>     <p>根容器是<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>内的概念，当某个<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>首次创建并展示时，根据<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>的特性，焦点会立即被该<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>抢占。此时，该<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>所在焦点链的末端节点将成为默认焦点，而这个默认焦点通常位于该<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>的根容器上。</p>     <p>在缺省状态下，<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>的默认焦点位于其根容器上，但开发者可以通过defaultFocus属性来自定义这一行为。</p>     <p>当焦点位于根容器时，首次按下Tab键不仅会使焦点进入激活状态，还会根据<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#焦点传递规则">焦点传递规则</a>进行传递。</p>    </div>    <div class="tiledSection">     <h3 id="焦点传递规则">焦点传递规则<i class="anchor-icon anchor-icon-link" anchorid="焦点传递规则" tips="复制节点链接"></i></h3>          <p>焦点传递是指当用户首次激活应用焦点系统时，焦点如何从根节点逐级向下传递到具体组件的过程。</p>     <p>在焦点链上的组件，都会处于获焦状态。同时组件在获焦时，会继续向下递归传递获焦状态，每次传递给第一个子组件，直到叶子节点。</p>       <p>根据走焦的触发方式，可以分为主动走焦和被动走焦。</p>     <p><strong>主动走焦</strong></p>     <p>指开发者/用户主观行为导致的焦点移动，包括：使用外接键盘的按键走焦（Tab键/Shift+Tab键/方向键）、使用requestFocus申请焦点、clearFocus清除焦点、focusOnTouch点击申请焦点等接口导致的焦点转移。</p>     <ul>      <li>按键走焦</li>     </ul>     <ol>      <li>前提：当前应用需处于焦点激活态。</li>      <li>范围限制：按键走焦仅在当前获得焦点的层级页面内进行，具体参见“层级页面”中的“走焦范围限制”部分。</li>      <li>       <p>按键类型：</p>       <p>Tab键：遵循Z字型遍历逻辑，完成当前范围内所有叶子节点的遍历，到达当前范围内的最后一个组件后，继续按下Tab键，焦点将循环至范围内的第一个可获焦组件，实现循环走焦。</p>       <p>Shift+Tab键：与Tab键具有相反的焦点转移效果。</p>       <p>方向键（上、下、左、右）：遵循十字型移动策略，在单层容器中，焦点的转移由该容器的特定走焦算法决定。若算法判定下一个焦点应落在某个容器组件上，系统将采用中心点距离优先的算法来进一步确定容器内的目标子节点。</p></li>      <li>走焦算法：每个可获焦的容器组件都有其特定的走焦算法，用于定义焦点转移的规则。</li>      <li>子组件优先：当子组件处理按键走焦事件，父组件将不再介入。</li>     </ol>     <ul>      <li>       <p>requestFocus</p>       <p>详见<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#主动获焦失焦">主动获焦失焦</a>，可以主动将焦点转移到指定组件上。</p>       <p>不可跨窗口或跨ArkUI实例申请焦点，但可以跨层级页面申请焦点。</p></li>      <li>       <p>clearFocus</p>       <p>详见<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/arkts-apis-uicontext-focuscontroller#clearfocus12" target="_blank">clearFocus</a>，会清除当前层级页面中的焦点，最终焦点停留在根容器上。</p></li>      <li>       <p>focusOnTouch</p>       <p>详见<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-attributes-focus#focusontouch9" target="_blank">focusOnTouch</a>，使绑定组件具备点击后获得焦点的能力。若组件本身不可获焦，则此功能无效。若绑定的是容器组件，点击后优先将焦点转移给上一次获焦的子组件，否则转移给第一个可获焦的子组件。</p></li>     </ul>     <p><strong>被动走焦</strong></p>     <p>被动走焦是指组件焦点因系统或其他操作而自动转移，无需开发者直接干预，这是焦点系统的默认行为。</p>     <p>目前会被动走焦的机制有：</p>     <ul>      <li>组件删除：当处于焦点状态的组件被删除时，焦点框架首先尝试将焦点转移到相邻的兄弟组件上，遵循先向后再向前的顺序。若所有兄弟组件均不可获焦，则焦点将释放，并通知其父组件进行焦点处理。</li>      <li>属性变更：若将处于焦点状态的组件的focusable或enabled属性设置为false，或者将visibility属性设置为不可见，系统将自动转移焦点至其他可获焦组件，转移方式与1中相同。</li>      <li><a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>切换：当发生<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>切换时，如从一个<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>跳转到另一个<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>，当前<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>的焦点将自动释放，新<a href="/consumer/cn/doc/harmonyos-guides/arkts-common-events-focus-event#基础概念">层级页面</a>可能会根据预设逻辑自动获得焦点。</li>      <li>Web组件初始化：对于Web组件，当其被创建时，若其设计需要立即获得焦点（如某些弹出框或输入框），则可能触发焦点转移至该Web组件，其行为属于组件自身的行为逻辑，不属于焦点框架的规格范围。</li>     </ul>    </div>    <div class="tiledSection">     <h3 id="走焦算法">走焦算法<i class="anchor-icon anchor-icon-link" anchorid="走焦算法" tips="复制节点链接"></i></h3>          <p>在焦点管理系统中，每个可获焦的容器都配备有特定的走焦算法，这些算法定义了当使用Tab键、Shift+Tab键或方向键时，焦点如何从当前获焦的子组件转移到下一个可获焦的子组件。</p>     <p>容器采用何种走焦算法取决于其UX（用户体验）规格，并由容器组件进行适配。目前，焦点框架支持三种走焦算法：线性走焦、投影走焦和自定义走焦。</p>     <p><strong>线性走焦算法</strong></p>     <p>线性走焦算法是默认的走焦策略，它基于容器中子节点在节点树中的挂载顺序进行走焦，常用于单方向布局的容器，如Row、Column和Flex容器。运行规则如下：</p>     <ul>      <li>顺序依赖：走焦顺序完全基于子节点在节点树中的挂载顺序，与它们在界面上的实际布局位置无关。</li>      <li>Tab键走焦：使用Tab键时，焦点将按照子节点的挂载顺序依次遍历。</li>      <li>方向键走焦：当使用与容器定义方向垂直的方向键时，容器不接受该方向的走焦请求。例如，在横向的Row容器中，无法使用方向键进行上下移动。</li>      <li>边界处理：当焦点位于容器的首尾子节点时，容器将拒绝与当前焦点方向相反的方向键走焦请求。例如，焦点在一个横向的Row容器的第一个子节点上时，该容器无法处理方向键左的走焦请求。</li>     </ul>     <div _ngcontent-abf-c106="" class="highlight-div"><div _ngcontent-abf-c106=""
