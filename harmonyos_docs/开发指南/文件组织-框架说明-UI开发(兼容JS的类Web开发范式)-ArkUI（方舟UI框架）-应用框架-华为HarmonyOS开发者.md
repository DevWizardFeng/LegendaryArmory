tips="复制节点链接"></i></h2><p>应用资源可通过绝对路径或相对路径的方式进行访问，绝对路径以"/"开头，相对路径以"./"或"../"。具体访问规则如下：</p> <ul><li><p>引用代码文件，推荐使用相对路径，比如：../common/utils.js。</p>  </li><li><p>引用资源文件，推荐使用绝对路径。比如：/common/xxx.png。</p>  </li><li><p>公共代码文件和资源文件推荐放在common下，通过以上两条规则进行访问。</p>  </li><li><p>CSS样式文件中通过url()函数创建&lt;url&gt;数据类型，如：url(/common/xxx.png)。</p>  </li></ul> <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content"> <p>当代码文件A需要引用代码文件B时：</p>  <ul><li><p>如果代码文件A和文件B位于同一目录，则代码文件B引用资源文件时可使用相对路径，也可使用绝对路径。</p>  </li><li><p>如果代码文件A和文件B位于不同目录，则代码文件B引用资源文件时必须使用绝对路径。因为Webpack打包时，代码文件B的目录会发生变化。</p>  </li><li><p>在js文件中通过数据绑定的方式指定资源文件路径时，必须使用绝对路径。</p>  </li></ul>  </div></div></div> </div>  <div
