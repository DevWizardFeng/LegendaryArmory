href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/module-configuration-file">module.json5配置文件</a>中的<a <p>系统将调用方传入的want参数（包含action、entities、uri、type和parameters属性）与已安装待匹配应用组件的skills配置（包含actions、entities、uris和type属性）进行匹配。当want参数五个属性匹配均未配置，隐式匹配失败。</p>     <ul>      <li>当parameters中的linkFeature字段取值不为空时，系统将优先进行linkFeature匹配。       <ul>        <li>如果linkFeature匹配成功，并且want中配置了uri或type，则继续匹配uri和type属性，均匹配成功则隐式匹配成功；否则，匹配失败。如果want中未配置uri和type, 则隐式匹配成功。</li>        <li>如果linkFeature匹配失败，则不进行后续属性匹配，匹配失败。</li>       </ul></li>      <li>当parameters中的linkFeature未配置或取值为空时，只有当action、entities、uri和type四个属性均匹配通过时，此应用才会被应用选择器展示给用户进行选择。</li>     </ul>    </div>    <div <p>调用方传入的want参数的action不为空，待匹配应用组件的skills配置中的actions不为空且包含调用方传入的want参数的action，则action匹配成功。</p></li>      <li>       <p>调用方传入的want参数的action不为空，待匹配应用组件的skills配置中的actions不为空且不包含调用方传入的want参数的action，则action匹配失败。</p></li>     </ul>     <p><strong>图1</strong> want参数的action匹配规则</p>     <p><span><img originheight="412" tips="复制节点链接"></i></h3>          <p>调用方传入的want参数中设置uri和type参数发起启动应用组件的请求，系统会遍历当前系统已安装的组件列表，并逐个匹配待匹配应用组件的skills配置中的uris数组，如果待匹配应用组件的skills配置中的uris数组中只要有一个可以匹配调用方传入的want参数中设置的uri和type即为匹配成功。</p>     <p>实际应用中，uri和type共存在四种情况，下面将讲解四种情况的具体匹配规则：</p>     <ul>      <li>       <p>调用方传入的want参数的uri和type都为空。</p>       <ol>        <li>如果待匹配应用组件的skills配置中的uris数组为空，匹配成功。</li>        <li>如果待匹配应用组件的skills配置中的uris数组中存在uri的scheme和type都为空的元素，匹配成功。</li>        <li>除以上两种情况，其他情况均为匹配失败。</li>       </ol></li>      <li>       <p>调用方传入的want参数的uri不为空，type为空。</p>       <ol>        <li>如果待匹配应用组件的skills配置中的uris数组为空，匹配失败。</li>        <li>如果待匹配应用组件的skills配置中的uris数组存在一条数据<a href="/consumer/cn/doc/harmonyos-guides/explicit-implicit-want-mappings#uri匹配规则">uri匹配</a>成功且type为空，则匹配成功，否则匹配失败。</li>        <li>如果前两条均匹配失败，并且传入的uri为文件路径uri，则根据文件后缀获取文件的MIME类型，如果该类型与skills文件中配置的type相匹配，则匹配成功。</li>       </ol></li>      <li>       <p>调用方传入的want参数的uri为空，type不为空。</p>       <ol>        <li>如果待匹配应用组件的skills配置中的uris数组为空，匹配失败。</li>        <li>如果待匹配应用组件的skills配置中的uris数组存在一条数据uri的scheme为空且<a href="/consumer/cn/doc/harmonyos-guides/explicit-implicit-want-mappings#type匹配规则">type匹配</a>成功，则匹配成功，否则匹配失败。</li>       </ol></li>      <li>       <p>调用方传入的want参数的uri和type都不为空，如下图所示。</p>       <ol>        <li>如果待匹配应用组件的skills配置中的uris数组为空，匹配失败。</li>        <li>如果待匹配应用组件的skills配置中的uris数组存在一条数据<a href="/consumer/cn/doc/harmonyos-guides/explicit-implicit-want-mappings#uri匹配规则">uri匹配</a>和<a href="/consumer/cn/doc/harmonyos-guides/explicit-implicit-want-mappings#type匹配规则">type匹配</a>需要均匹配成功，则匹配成功，否则匹配失败。</li>       </ol></li>     </ul> width="799" height="455"></span></p>    </div>    <div class="tiledSection">     <h3 id="uri匹配规则">uri匹配规则<i class="anchor-icon anchor-icon-link" anchorid="uri匹配规则" tips="复制节点链接"></i></h3>          <p>具体的匹配规则如下：</p>     <ul>      <li>       <p>如果s_uri的scheme为空，当w_uri为空时匹配成功，否则匹配失败。</p></li>      <li>       <p>如果s_uri的host为空，当w_uri和s_uri的scheme相同时匹配成功，否则匹配失败。</p></li>      <li>       <p>如果s_uri的port为空，当w_uri和s_uri中的scheme和host相同时匹配成功，否则匹配失败。</p></li>      <li>       <p>如果s_uri的path、pathStartWith和pathRegex都为空，当w_uri和s_uri中的scheme，host和port相同时匹配成功，否则匹配失败。</p></li>      <li>       <p>如果s_uri的path不为空，当w_uri和s_uri<strong>全路径表达式</strong>相同时匹配成功，否则继续进行pathStartWith的匹配。</p></li>      <li>       <p>如果s_uri的pathStartWith不为空，当w_uri包含s_uri<strong>前缀表达式</strong>时匹配成功，否则继续进行pathRegex的匹配。</p></li>      <li>       <p>如果s_uri的pathRegex不为空，当w_uri满足s_uri<strong>正则表达式</strong>时匹配成功，否则匹配失败。</p></li>     </ul>     <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">       <p>待匹配应用组件的skills配置的uris中scheme、host、port、path、pathStartWith和pathRegex属性拼接，如果依次声明了path、pathStartWith和pathRegex属性时，uris将分别拼接为如下四种表达式：</p>       <ul>        <li><strong>前缀uri表达式</strong>：当配置文件只配置scheme，或者只配置scheme和host，或者只配置scheme，host和port时，参数传入以配置文件为前缀的Uri         <ul>          <li>scheme://</li>          <li>scheme://host</li>          <li>scheme://host:port</li>         </ul></li>        <li><strong>全路径表达式</strong>：scheme://host:port/path</li>        <li><strong>前缀表达式</strong>：scheme://host:port/pathStartWith</li>        <li><strong>正则表达式</strong>：scheme://host:port/pathRegex</li>       </ul>       <p>系统应用预留uri的scheme统一以ohos开头，例如ohosclock://。三方应用组件配置的uri不能与系统应用重复，否则会导致无法通过该uri拉起三方应用组件。</p>      </div></div></div>  <p>如果w_type最后一个字符为通配符*，如prefixType/*，则当s_type包含prefixType/时匹配成功，否则匹配失败。</p></li>     </ul>    </div>    <div class="tiledSection">     <h3 id="linkfeature匹配规则">linkFeature匹配规则<i class="anchor-icon <p>本章节所述的linkFeature匹配规则适用于want参数中的parameters包含linkFeature键，且对应取值不为空的场景。</p>      </div></div></div>     <p>将调用方传入的want参数的parameters与待匹配应用组件的skills配置中的uris进行匹配。为了简化描述,
