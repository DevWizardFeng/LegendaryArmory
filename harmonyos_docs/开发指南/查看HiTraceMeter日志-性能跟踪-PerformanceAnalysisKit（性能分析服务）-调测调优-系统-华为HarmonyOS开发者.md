anchor-icon-link" anchorid="用户态trace格式说明" tips="复制节点链接"></i></h2>          <p>开发者可以通过<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/hitrace">hitrace</a>命令采集用户通过HiTraceMeter接口生成的用户态trace日志，从而对应用进行性能分析。本章节将详细介绍用户态trace的格式和扩展规则，包括同步、异步和整数值跟踪等类型。每种trace的字段组成、含义及与打点入参的关系都会逐一说明，并提供<a href="/consumer/cn/doc/harmonyos-guides/hitracemeter-view#各场景trace日志样例">具体样例</a>供参考。</p>     <p>API version 19及以上版本的用户态trace格式参考<a href="/consumer/cn/doc/harmonyos-guides/hitracemeter-view#当前版本用户态trace格式">当前版本用户态trace格式</a>，API version 19以下版本的用户态trace格式参考<a href="/consumer/cn/doc/harmonyos-guides/hitracemeter-view#api-version-19前用户态trace格式">API version 19前用户态trace格式</a>。</p>    </div>    <div class="tiledSection">     <h3 id="当前版本用户态trace格式" class="firsth2">当前版本用户态trace格式<i class="anchor-icon anchor-icon-link" anchorid="当前版本用户态trace格式" tips="复制节点链接"></i></h3>          <p><strong>同步跟踪</strong></p>     <ul>      <li>       <p>开始trace格式：[类型标识符]|[进程号]|H:[分布式跟踪链路ID]#[内容]|[级别与所属Tag集合]|[自定义键值对]</p></li>      <li>       <p>结束trace格式：[类型标识符]|[进程号]|[级别与所属Tag集合]</p></li>     </ul>     <p>一条完整的同步跟踪包括一个开始trace和一个结束trace，开始trace的类型标识符固定为B，结束trace的类型标识符固定为E。</p>     <p><strong>异步跟踪</strong></p>     <ul>      <li>       <p>起始trace格式：[类型标识符]|[进程号]|H:[分布式跟踪链路ID]#[内容]|[异步任务ID]|[级别与所属Tag集合]|[自定义聚类名称]|[自定义键值对]</p></li>      <li>       <p>结束trace格式：[类型标识符]|[进程号]|H:[分布式跟踪链路ID]#[内容]|[异步任务ID]|[级别与所属Tag集合]</p></li>     </ul>     <p>一条完整的异步跟踪也同样包含一个开始trace和一个结束trace，开始trace的类型标识符固定为S，结束trace的类型标识符固定为F。</p>     <p><strong>整数值跟踪</strong></p>     <ul>      <li>trace格式：[类型标识符]|[进程号]|H:[分布式跟踪链路ID]#[内容]|[整数值]|[级别与所属Tag集合]</li>     </ul>     <p>一条整数值跟踪trace仅包含一条独立trace，其中trace的类型标识符固定为C。</p>     <div><div class="hw-editor-tip warn"><div class="title">注意</div><div class="content">       <ol>        <li>一条有效且完整的同步跟踪trace与异步跟踪trace，均包含一条起始trace和一条结束trace，开发者需要保证接口调用的匹配性，否则会导致性能跟踪异常问题。具体可参考ArkTS接口<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-hitracemeter" target="_blank">@ohos.hiTraceMeter (性能打点)</a>或C++接口<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/capi-trace-h" target="_blank">capi-trace-h</a>。</li>        <li>用户态trace总长度限制512字符，超过部分将被截断。</li>       </ol>      </div></div></div>    </div>    <div class="tiledSection">     <h3 id="字段说明">字段说明<i class="anchor-icon anchor-icon-link" anchorid="字段说明" tips="复制节点链接"></i></h3>          <p>本小节将详细介绍用户态trace的各个字段，包括字段组成、含义，字段与打点入参之间的关系，使用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-hitracemeter" target="_blank">@ohos.hiTraceMeter (性能打点)</a>作为参考。</p>     <ul>      <li>       <p><strong>类型标识符</strong></p>       <p>类型标识符与调用的<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-hitracemeter" target="_blank">@ohos.hiTraceMeter (性能打点)</a>具体接口相关，其映射关系如下：</p>       <ul>        <li>B：开启一个同步时间片跟踪事件。</li>        <li>E：结束一个同步时间片跟踪事件。</li>        <li>S：开启一个异步时间片跟踪事件。</li>        <li>F：结束一个异步时间片跟踪事件。</li>        <li>C：整数跟踪事件。</li>       </ul></li>      <li>       <p><strong>分布式跟踪链路ID</strong></p>       <p>开启HiTraceChain（分布式跟踪）时，会将<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/capi-hitrace-hitraceid" target="_blank">HiTraceId</a>核心信息记录在trace打点中，格式为[ChainId,SpanId,ParentSpanId]，结尾使用#分隔符与内容字段区分。</p>       <p>未开启HiTraceChain（分布式跟踪）时，打点不包含该字段。</p>       <ul>        <li>ChainId：HiTraceId的跟踪链标识。</li>        <li>SpanId：HiTraceId的分支标识。</li>        <li>ParentSpanId：HiTraceId的父分支标识。</li>       </ul>       <p>HiTraceChain具体使用指导请参考<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/hitracechain-guidelines-arkts">使用HiTraceChain打点（ArkTS）</a>与<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/hitracechain-guidelines-ndk">使用HiTraceChain打点（C/C++）</a>。</p></li>      <li>       <p><strong>内容</strong></p>       <p>表示打点名称或整数变量名。对应<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-hitracemeter" target="_blank">@ohos.hiTraceMeter (性能打点)</a>接口的name入参。</p></li>      <li>       <p><strong>异步任务ID</strong></p>       <p>和内容字段合并用于匹配相同的异步跟踪trace的开始与结束打点。对应<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-hitracemeter" target="_blank">@ohos.hiTraceMeter (性能打点)</a>接口的taskId入参。</p></li>      <li>       <p><strong>整数值</strong></p>       <p>仅在整数值跟踪trace中包含，为跟踪的整数值。对应<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-hitracemeter" target="_blank">@ohos.hiTraceMeter (性能打点)</a>接口的count入参。</p></li>      <li>       <p><strong>级别与所属Tag集合</strong></p>       <p>本字段由级别和所属Tag集合两部分信息组成。</p>       <p>通过打点接口传入的级别信息，该部分为以下字符之一：D、I、C、M。对应<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-hitracemeter" target="_blank">@ohos.hiTraceMeter (性能打点)</a>接口的level入参，详细介绍及映射关系参考<a href="/consumer/cn/doc/harmonyos-guides/hitracemeter-view#hitraceoutputlevel">HiTraceOutputLevel</a>。</p>       <p>所属Tag集合表示trace打点所属Tag类别，每类用两位数字表示，可包含多个从小到大排列的两位数字。</p>       <p>例如B|1314|H:TestFunction|I3062，其中I表示该条打点级别为Info，3062表示该条打点所属Tag集合，根据Tag名称与值的映射关系，参考<a href="/consumer/cn/doc/harmonyos-guides/hitracemeter-view#hitracemeter-tag">HiTraceMeter Tag</a>可知30对应ohosTag，62对应appTag。</p>       <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">         <ol>          <li>           <p>对于API version 19及之后接口，支持自定义输出级别；对于API version 18及之前接口，trace打点默认为COMMERCIAL级别打点。</p></li>          <li>           <p>系统侧存在一条用户态trace隶属于多个Tag的情况，ArkTS接口<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-hitracemeter" target="_blank">@ohos.hiTraceMeter (性能打点)</a>和C++接口<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/capi-trace-h" target="_blank">capi-trace-h</a>提供的应用侧用户态trace打点功能无法指定所属Tag集合，对应Tag分类固定为app，字段值为62。</p></li>         </ol>        </div></div></div></li>      <li>       <p><strong>自定义聚类名称</strong></p>       <p>可用于开发者自定义的聚类操作。对应<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-hitracemeter" target="_blank">@ohos.hiTraceMeter (性能打点)</a>接口的customCategory入参。该字段值可能为空，从而使得日志格式发生变化，详见<a href="/consumer/cn/doc/harmonyos-guides/hitracemeter-view#字段为空情况说明">字段为空情况说明</a>。</p>       <p>在trace可视化工具中，同一进程中相同聚类名称的异步打点放在同一泳道上进行展示。</p></li>      <li>       <p><strong>自定义键值对</strong></p>       <p>格式为key=value，多个键值对使用逗号分隔，例如key1=value1,key2=value2。对应<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-hitracemeter" target="_blank">@ohos.hiTraceMeter (性能打点)</a>接口的customArgs入参。该字段值可能为空，从而使得日志格式发生变化，详见<a href="/consumer/cn/doc/harmonyos-guides/hitracemeter-view#字段为空情况说明">字段为空情况说明</a>。</p>       <p>在trace可视化工具中，对满足格式要求的自定义键值对，以键值对形式展示解析结果。</p></li>     </ul>    </div>    <div class="tiledSection"> <div class="tiledSection">     <h3 id="字段为空情况说明">字段为空情况说明<i class="anchor-icon anchor-icon-link" anchorid="字段为空情况说明" tips="复制节点链接"></i></h3>          <p>如果用户态trace格式中间字段对应接口参数传入内容为空，用户态trace格式将保留|分隔符，如果尾部字段为空，用户态trace格式将不保留|分隔符。以<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-hitracemeter#hitracemeterstartasynctrace19" target="_blank">hiTraceMeter.startAsyncTrace()</a>和<a
