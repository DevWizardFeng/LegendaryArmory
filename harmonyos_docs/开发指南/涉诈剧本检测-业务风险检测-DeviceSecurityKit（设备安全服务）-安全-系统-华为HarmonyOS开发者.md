<h1 _ngcontent-dbk-c119="" class="doc-title ng-star-inserted" title="涉诈剧本检测"> 涉诈剧本检测 </h1>

<div _ngcontent-dbk-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <div class="tiledSection">     <h2 id="section1061418367582">场景介绍<i class="anchor-icon anchor-icon-link" anchorid="section1061418367582" tips="复制节点链接"></i></h2>          <p>从5.0.0(12) 版本开始，新增支持涉诈剧本检测。</p>     <p>金融支付类应用在用户转账、支付前，通过调用Device Security kit的detectFraudRisk接口，检测用户是否受到欺诈威胁。该接口返回一个风险分，以及涉诈行为的线索，例如，接收到涉诈引导信息、设备有被操控风险等，应用可以根据风险分及线索，进行有效提示或拦截。</p>    </div>    <div class="tiledSection">     <h2 id="section097084185914">约束与限制<i class="anchor-icon anchor-icon-link" anchorid="section097084185914" tips="复制节点链接"></i></h2>          <p>每个应用在每个设备上每天最多可以调用10次。</p>    </div>    <div class="tiledSection">     <h2 id="section123619227598">业务流程<i class="anchor-icon anchor-icon-link" anchorid="section123619227598" tips="复制节点链接"></i></h2>          <p><span><img originheight="676" originwidth="1124" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20251211115029.94970344518873304448604468263367:50001231000000:2800:AA13597E8D8C8D4D31E83AC8DB5721AC454717B3F0F00FC5B2540E59D6EB2BFA.png" width="920" height="553.3096085409253"></span></p>    </div>    <p><strong>流程说明：</strong></p>    <ol>     <li><strong>开发者应用获取nonce。</strong>      <p>在调用detectFraudRisk接口时，开发者必须传入一个随机生成的nonce值。在检测结果中会包含这个nonce值，您可以通过校验这个nonce值来确定返回结果能够对应您的请求，并且没有被重放攻击。</p>      <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">        <ul>         <li>nonce值必须为24至80字节之间。</li>         <li>推荐的做法是，每次请求都从服务器随机生成新的nonce值。</li>        </ul>       </div></div></div></li>     <li><strong>开发者应用调用detectFraudRisk接口，发起涉诈剧本检测请求。</strong>      <p>Device Security Kit收到请求后，首先采集当前设备涉诈风险线索数据，然后将线索数据与nonce一起发送到Device Security服务器做检测，最后通过detectFraudRisk接口的返回值将检测结果传递给开发者应用。</p></li>
