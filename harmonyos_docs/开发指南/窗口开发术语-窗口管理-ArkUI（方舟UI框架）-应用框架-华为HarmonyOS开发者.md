<h1 _ngcontent-omo-c119="" class="doc-title ng-star-inserted" title="窗口开发术语"> 窗口开发术语 </h1>

<div _ngcontent-omo-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <div class="tiledSection">     <h2 id="窗口类型">窗口类型<i class="anchor-icon anchor-icon-link" anchorid="窗口类型" tips="复制节点链接"></i></h2>          <p>窗口有系统窗口、应用窗口两种分类。</p>    </div>    <div class="tiledSection">     <h3 id="系统窗口" class="firsth2">系统窗口<i class="anchor-icon anchor-icon-link" anchorid="系统窗口" tips="复制节点链接"></i></h3>          <p>系统窗口指完成系统特定功能的窗口。如音量条、壁纸、通知栏、状态栏、导航栏等。</p>     <p>系统窗口仅系统应用可用。</p>    </div>    <div class="tiledSection">     <h3 id="应用窗口">应用窗口<i class="anchor-icon anchor-icon-link" anchorid="应用窗口" tips="复制节点链接"></i></h3>          <p>应用窗口区别于系统窗口，指与应用显示相关的窗口，用来显示应用的内容。根据管理方式和用途的不同，应用窗口又可以分为主窗口、辅助窗口两种类型。</p>     <ul>      <li>       <p>主窗口</p>       <p>主窗口由UIAbility创建时默认创建，会在“任务管理界面”中以一个独立的任务卡片显示，用于显示应用UIAbility主界面。</p></li>      <li>       <p>辅助窗口</p>       <p>辅助窗口由应用自行管理创建和销毁，不会在“任务管理界面”中以一个独立的任务卡片显示，可以用于显示应用的辅助内容，例如弹窗等。</p>       <p>辅助窗口包括子窗口、全局悬浮窗、模态窗口、画中画和闪控球。</p></li>     </ul>    </div>    <div class="tiledSection">     <h2 id="悬浮窗">悬浮窗<i class="anchor-icon anchor-icon-link" anchorid="悬浮窗" tips="复制节点链接"></i></h2>          <p>悬浮窗分为智慧多窗悬浮窗和全局悬浮窗。</p>     <ul>      <li>       <p>智慧多窗悬浮窗是指设备屏幕上悬浮的、非全屏的应用窗口。</p>       <p>一般用于在已有全屏任务运行的基础上，临时处理另一个任务，或短时间多任务并行使用。如浏览网页的同时回复消息。</p>       <p>相关参考：<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/multi-window-intro" target="_blank">智慧多窗简介</a>、<a href="https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-multi-window-practice" target="_blank">智慧多窗最佳实践</a>。</p></li>      <li>       <p>全局悬浮窗是指一种特殊的应用辅助窗口，具备在应用主窗口和对应Ability退至后台后仍然可以在前台显示的能力。</p>       <p>全局悬浮窗可以用于应用退至后台后，使用小窗继续显示UI，例如音乐应用用于显示桌面歌词等。</p>       <p>应用在创建全局悬浮窗前，需要申请对应的权限。</p></li>     </ul>     <p>相关参考：<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/application-window-stage#设置全局悬浮窗受限开放">设置全局悬浮窗</a>。</p>    </div>    <div class="tiledSection">     <h2 id="自由窗口">自由窗口<i class="anchor-icon anchor-icon-link" anchorid="自由窗口" tips="复制节点链接"></i></h2>          <p>自由窗口是一种允许用户在同一屏幕上以自由大小、位置显示的窗口状态。自由窗口支持拖拽、缩放和分屏组合，从而实现多任务处理。</p>     <p>自由窗口按照打开或者获取焦点的顺序在Z轴层叠排布。当自由窗口被点击或触摸时，将导致其Z轴高度提升，并获取焦点。</p>     <p>启动新的自由窗口时，默认以一定间距在上一个窗口的右下方层叠显示。</p>     <p>每个自由窗口默认会在窗口上方显示窗口标题栏，标题栏左侧显示应用图标，右侧显示三键控制按钮（窗口最大化/还原、窗口最小化和关闭窗口），且窗口标题栏支持额外的<a href="https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-multi-device-window-immersive#section1477255312219" target="_blank">沉浸式配置</a>。</p>     <p>自由窗口可以通过拖动窗口边缘调节窗口大小，可以通过拖动标题栏移动窗口位置。</p>     <p><span><img originheight="1600" originwidth="2400" width="920" height="613.3333333333334"></span></p>     <p>当前设备支持情况：</p>     <ul>      <li><strong>2in1设备</strong>：2in1设备上的窗口，默认为自由窗口。</li>      <li><strong>Tablet设备</strong>：部分Tablet设备，支持开启<a href="/consumer/cn/doc/harmonyos-guides/window-terminology#自由多窗模式">自由多窗模式</a>（通过下拉控制中心，点击“自由多窗”按钮开启），开启此模式后，应用窗口默认为自由窗口。</li>      <li><strong>Phone设备</strong>：部分Phone设备，支持开启<a href="/consumer/cn/doc/harmonyos-guides/window-terminology#自由多窗模式">自由多窗模式</a>（通过下拉控制中心，点击“自由多窗”按钮开启），开启此模式后，应用窗口默认为自由窗口。</li>     </ul>    </div>    <div class="tiledSection">     <h3 id="自由多窗模式" class="firsth2">自由多窗模式<i class="anchor-icon anchor-icon-link" anchorid="自由多窗模式" tips="复制节点链接"></i></h3>          <p>自由多窗模式是一种支持用户在移动设备上进行多任务处理的交互方式。</p>     <p>自由多窗下，允许用户在一块屏幕上同时显示多个应用窗口。此时的应用窗口为<a href="/consumer/cn/doc/harmonyos-guides/window-terminology#自由窗口">自由窗口</a>。</p>     <p>部分Tablet设备上，可通过下拉控制中心，点击“自由多窗”按钮开启自由多窗。</p>     <p>部分Phone设备上，可通过下拉控制中心，点击“自由多窗”按钮开启自由多窗。</p>     <p><span><img width="920" height="357.8567748583205"></span></p>    </div>    <div class="tiledSection">     <h2 id="沉浸式布局">沉浸式布局<i class="anchor-icon anchor-icon-link" anchorid="沉浸式布局" tips="复制节点链接"></i></h2>
