<h1 _ngcontent-ent-c119="" class="doc-title ng-star-inserted" title="系统完整性增强检测"> 系统完整性增强检测 </h1>

<div _ngcontent-ent-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <div class="tiledSection">     <h2 id="section1880231055910">场景介绍<i class="anchor-icon anchor-icon-link" anchorid="section1880231055910" tips="复制节点链接"></i></h2>          <p>应用通过调用Device Security Kit的checkSysIntegrityEnhanced接口获取系统完整性增强检测结果，用于判断设备环境是否安全，比如是否被越狱、非真实设备等。</p>     <p>应用可以根据检测结果评估如何进行业务操作。</p>    </div>    <div class="tiledSection">     <h2 id="section9684181513312">约束与限制<i class="anchor-icon anchor-icon-link" anchorid="section9684181513312" tips="复制节点链接"></i></h2>          <ul>      <li>系统完整性增强检测能力支持Phone、Tablet、PC/2in1、Wearable设备。</li>      <li>每个应用在每个设备上每天最多可以调用1万次接口、每分钟最多可以调用5次接口；每个设备上最多支持5个并发调用。</li>     </ul>    </div>    <div class="tiledSection">     <h2 id="section1818915573916">业务流程<i class="anchor-icon anchor-icon-link" anchorid="section1818915573916" tips="复制节点链接"></i></h2>          <p><span><img height="537.9609912070345" originheight="866" originwidth="1481" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20251211115049.07347594633897315869094968919163:50001231000000:2800:8FE9F867A7A0B7D3C3B10F49741B7EDCE7B118610CA03905899D3EB51C6E30E8.png" title="点击放大" width="920"></span></p>     <p><strong>流程说明：</strong></p>    </div>    <p>1.<strong> 开发者应用获取nonce。</strong></p>    <p>在调用checkSysIntegrityEnhanced接口时，您必须传入一个随机生成的nonce值。在检测结果中会包含这个nonce值，您可以通过校验这个nonce值来确定返回结果能够对应您的请求，并且没有被重放攻击。</p>    <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">      <ul>       <li>nonce值必须为16至66字节之间，有效值为base64编码范围。</li>       <li>推荐的做法是，每次请求都从服务器随机生成新的nonce值。</li>      </ul>     </div></div></div>    <p>2.<strong> 开发者应用调用checkSysIntegrityEnhanced接口，发起系统完整性增强检测请求。</strong></p>    <p>Device Security Kit收到请求后，首先采集系统完整性增强检测数据，然后将检测数据与nonce一起发送到Device Security服务器做检测，最后通过checkSysIntegrityEnhanced接口的返回值将检测结果传递给开发者应用。</p>    <p>3.<strong> 当开发者应用发起业务请求时，在您的应用服务器中验证系统完整性增强检测结果。</strong></p>    <p>当系统完整性增强检测结果为false时，请进一步判断detail中的具体风险分类，您可以根据风险分类以及自身功能对安全的要求决定是否提醒用户。</p>    <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">      <ul>       <li>当前方案已经通过服务端与客户端相结合等措施进行安全风险消减，但系统完整性增强检测API无法消减所有的安全风险。</li>       <li>系统完整性增强检测结果可以用作系统整体安全的一个环节，需要考虑检测结果误报带来的风险以及给用户带来的影响，不建议将系统完整性增强检测结果作为判断当前设备是否安全的唯一依据，更好的做法是通过额外的步骤降低风险。</li>       <li>如果需要在应用中提醒用户，为了提升用户体验，建议采用友好的提示语，可参考：        <p>您的设备疑似存在风险或运行在不安全环境中，请谨慎使用xxx功能。</p></li>      </ul>     </div></div></div>    <div class="tiledSection">     <h2 id="section5906121818412">接口说明<i
