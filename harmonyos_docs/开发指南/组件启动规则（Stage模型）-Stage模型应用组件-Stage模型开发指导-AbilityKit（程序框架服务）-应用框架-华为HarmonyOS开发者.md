<h1 _ngcontent-bwv-c119="" class="doc-title ng-star-inserted" title="组件启动规则（Stage模型）"> 组件启动规则（Stage模型） </h1>

<div _ngcontent-bwv-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <p>启动组件是指一切启动或连接应用组件的行为：</p>    <ul>     <li>      <p>启动<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-app-ability-uiability" target="_blank">UIAbility</a>、ServiceExtensionAbility、DataShareExtensionAbility，如使用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-inner-application-uiabilitycontext#startability" target="_blank">startAbility()</a>、startServiceExtensionAbility()、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-inner-application-uiabilitycontext#startabilitybycall" target="_blank">startAbilityByCall()</a>、<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-inner-application-uiabilitycontext#openlink12" target="_blank">openLink()</a>等相关接口。</p></li>     <li>      <p>连接ServiceExtensionAbility、DataShareExtensionAbility，如使用<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-inner-application-uiabilitycontext#connectserviceextensionability" target="_blank">connectServiceExtensionAbility()</a>、createDataShareHelper()等相关接口。</p></li>    </ul>    <div class="tiledSection">     <h2 id="组件启动总体规则">组件启动总体规则<i class="anchor-icon anchor-icon-link" anchorid="组件启动总体规则" tips="复制节点链接"></i></h2>          <p>为了保证用户具有更好的使用体验，对以下几种易影响用户体验与系统安全的行为做了限制：</p>     <ul>      <li>       <p>后台应用任意弹框，如各种广告弹窗，影响用户使用。</p></li>      <li>       <p>后台应用相互唤醒，不合理的占用系统资源，导致系统功耗增加或系统卡顿。</p></li>      <li>       <p>前台应用任意跳转至其他应用，如随意跳转到其他应用的支付页面，存在安全风险。</p></li>     </ul>     <p>鉴于此，制订了一套组件启动规则，主要包括：</p>     <ul>      <li>       <p><strong>跨应用启动组件，需校验目标组件是否可以被其他应用调用。</strong></p>       <p>若目标组件exported字段配置为true，表示可以被其他应用调用；若目标组件exported字段配置为false，表示不可以被其他应用调用，还需进一步校验ohos.permission.START_INVISIBLE_ABILITY权限（该权限仅系统应用可申请）。组件exported字段说明可参考<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/module-configuration-file#abilities标签">abilities标签</a>。</p></li>      <li>       <p><strong>位于后台的UIAbility应用，启动组件需校验BACKGROUND权限ohos.permission.START_ABILITIES_FROM_BACKGROUND（该权限仅系统应用可申请）。</strong></p>       <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">         <ul>          <li>前后台应用的判断依据：若应用进程获焦或所属的UIAbility组件位于前台则判定为前台应用，否则为后台应用。</li>          <li>对于2in1和Tablet设备：           <ul>            <li>从API version 18开始，如果应用已创建在前台显示的悬浮窗，当该应用退至后台时，无需校验BACKGROUND权限也可以拉起其他Ability。</li>            <li>从API version 21开始，如果应用自身已经添加到状态栏，则当应用退至后台时，无需校验BACKGROUND权限也可以拉起自身的UIAbility。</li>           </ul></li>         </ul>        </div></div></div></li>      <li>       <p><strong>跨设备使用startAbilityByCall接口，需校验分布式权限ohos.permission.DISTRIBUTED_DATASYNC。</strong></p></li>     </ul>     <p>上述组件启动规则自API 9版本开始生效，新增规则生效版本在规则中单独说明。开发者需熟知组件启动规则，避免业务功能异常。启动组件的具体校验流程见下文。</p>    </div>    <div class="tiledSection">     <h2 id="同设备组件启动规则">同设备组件启动规则<i class="anchor-icon anchor-icon-link" anchorid="同设备组件启动规则"      <li>       <p>通过<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-inner-application-uiabilitycontext#startabilitybycall" target="_blank">startAbilityByCall</a>接口启动UIAbility。</p></li>     </ul>     <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">
