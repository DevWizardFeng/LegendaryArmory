<h1 _ngcontent-fsa-c119="" class="doc-title ng-star-inserted" title="组件扩展概述"> 组件扩展概述 </h1>

<div _ngcontent-fsa-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;"> <div><p>ArkUI使用@Builder装饰器为开发者提供代码精简方案。该装饰器通过模块化封装简化UI开发流程，并衍生出@BuilderParam、@LocalBuilder装饰器和wrapBuilder机制，构成可复用的UI结构体系。</p> <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content"> <p>@Builder装饰器是@BuilderParam装饰器、@LocalBuilder装饰器和wrapBuilder机制的基石。</p>  </div></div></div>  <div class="tiledSection"><h2 id="builder装饰器">@Builder装饰器<i class="anchor-icon anchor-icon-link" anchorid="builder装饰器" tips="复制节点链接"></i></h2><p><a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-builder">@Builder装饰器</a>专为构建模块化、可复用的UI结构而设计，其内部禁止定义状态变量和调用组件生命周期方法，仅支持通过参数与调用方进行数据交互。</p> </div>  <div class="tiledSection"><h2 id="localbuilder装饰器">@LocalBuilder装饰器<i class="anchor-icon anchor-icon-link" anchorid="localbuilder装饰器" tips="复制节点链接"></i></h2><p>在使用@Builder实现组件间数据传递时，开发者需注意组件层级关系。组件树中的父子关系可能与状态管理的父子关系产生不一致。为此，框架提供了<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-localbuilder">@LocalBuilder装饰器</a>来解决这一特定问题。</p> </div>  <div class="tiledSection"><h2 id="builderparam装饰器">@BuilderParam装饰器<i class="anchor-icon anchor-icon-link" anchorid="builderparam装饰器" tips="复制节点链接"></i></h2><p>当多个场景共用同一个@Builder函数时，若需要为特定场景扩展功能（类似slot占位符机制），可通过<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-builderparam">@BuilderParam装饰器</a>实现，该装饰器专门用于接收并封装@Builder函数。</p> </div>  <div class="tiledSection"><h2 id="wrapbuilder机制">wrapBuilder机制<i class="anchor-icon anchor-icon-link" anchorid="wrapbuilder机制" tips="复制节点链接"></i></h2><p>当页面中存在多个不同UI结构的全局@Builder函数时，维护成本较高。框架提供了<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-wrapbuilder">wrapBuilder</a>机制来简化代码维护工作。</p> </div> </div> <div></div></div>
