<h1 _ngcontent-ufg-c119="" class="doc-title ng-star-inserted" title="绑定全模态页面（bindContentCover）"> 绑定全模态页面（bindContentCover） </h1>

<div _ngcontent-ufg-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <p><a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-universal-attributes-modal-transition#bindcontentcover" target="_blank">全模态页面（bindContentCover）</a>是全屏模态形式的弹窗交互页面，完全覆盖底层父视图。适用于查看大图，全屏查看文稿等场景。</p>    <div class="tiledSection">     <h2 id="使用约束">使用约束<i class="anchor-icon anchor-icon-link" anchorid="使用约束" tips="复制节点链接"></i></h2>          <p>全模态页面本质上是弹窗类组件，其交互层级默认为应用内顶层。</p>     <p><a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-navigation" target="_blank">Navigation</a>导航转场时，新push的页面层级无法超出全模态，其效果仍然显示在模态页面之下。针对此类场景，建议将模态页面的内容迁移至转场页面中实现。例如，在上述情况下，可以使用NavDestination来替代拉起的模态页面，新push的页面层级低于全模态。</p>    </div>    <div class="tiledSection">     <h2 id="生命周期">生命周期<i class="anchor-icon anchor-icon-link" anchorid="生命周期" tips="复制节点链接"></i></h2>          <p>全模态页面提供了生命周期函数，用于通知应用程序该弹窗的生命周期状态。生命周期的触发顺序依次为：onWillAppear -&gt;
