<h1 _ngcontent-ysu-c119="" class="doc-title ng-star-inserted" title="自动续期订阅说明"> 自动续期订阅说明 </h1>

<div _ngcontent-ysu-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;">   <div>    <p>订阅是指用户在购买自动续期订阅商品后，可以在一段时间访问App的增值功能或内容，并且会在订阅周期结束后自动续期（自动购买下一期服务）的能力。如果期间用户取消订阅，则订阅在当期结束后将不再自动续期。</p>    <p><span><img height="563.9466" originheight="3053" originwidth="1440" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20251211114647.30073989811461724278850300473194:50001231000000:2800:2C0515459C5B1736C5EB07A8FD920F62DEF8AF0697D6B75C27488FD3BFD7CC37.png" title="点击放大" width="266"></span></p>    <div class="tiledSection">     <h2 id="section167197586468">配置自动续期订阅商品<i class="anchor-icon anchor-icon-link" anchorid="section167197586468" tips="复制节点链接"></i></h2>         </div>    <div class="tiledSection">     <h3 id="section1986219389113" class="firsth2">订阅组<i class="anchor-icon anchor-icon-link" anchorid="section1986219389113" tips="复制节点链接"></i></h3>          <p>自动续期订阅商品由<strong>订阅组</strong>来管理维护，所以在创建自动续期订阅商品前，需要先创建订阅组，并在创建自动续期订阅商品时指定商品所在的订阅组。</p>     <ul>      <li>订阅组用于承载同类型商品的管理，一个订阅组可以包含多个自动续期订阅商品。</li>      <li>同一个订阅组只有一个商品是处于生效状态。当商品处于生效状态时，开发者需要为用户发放权益。</li>     </ul>     <p>订阅组机制为开发者提供了便捷方式，让商品服务大致相同但有细微差别的需求得以快速实现。</p>    </div>    <div class="tiledSection">     <h3 id="section71731035317">续费周期<i class="anchor-icon anchor-icon-link" anchorid="section71731035317" tips="复制节点链接"></i></h3>          <p>当前自动续期订阅商品的续费周期可选值有1周、30天、31天、1个月、2个月、3个月、6个月和12个月，具体配置方法请参见<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/iap-config-product#section49178516208">配置商品信息</a>。</p>    </div>    <div class="tiledSection">     <h3 </div></div></div>    </div>    <div class="tiledSection">     <h2 id="section15231185754812">提供优惠<i class="anchor-icon anchor-icon-link" anchorid="section15231185754812" tips="复制节点链接"></i></h2>          <p>为了提供更有吸引力的自动续期订阅商品购买和挽留，自动续期订阅商品提供了三种促销类型。</p>     <ul>      <li id="li1088435316541"><strong>推介促销（新用户促销）</strong>       <p>适用于自动续期订阅商品的促销价格。开发者可以指定优惠持续时间和类型（免费试用、按周期扣费），可用于吸引新用户。</p>       <p>推介促销规则：</p>       <ol>        <li>用户对每个订阅组至多享受一次推介促销。如果一个订阅组里的多个商品均配置了推介促销活动，在用户尝试切换该订阅组的其他商品时，用户只能享受在该订阅组第一次遇到的推介促销。当切换订阅时在该订阅组再次遇到推介促销，将不再享有推介促销资格。</li>        <li>推介促销价享受的触发方式是新订阅、恢复订阅、切换订阅三种情况。推介促销的享受资格不受订阅方式影响，无论是新发起的订阅，恢复订阅，还是切换订阅等，均遵从“用户对每个订阅组至多享受一次推介促销”的原则，只要没有享受过该订阅组的推介促销，就可以享受推介促销。</li>       </ol></li>     </ul>     <ul>      <li id="li1264912560544"><strong>优惠促销（自定义人群促销）</strong>       <p>可以针对用户群体、优惠地域进行自定义选择，支持开发者进行个性化的优惠活动配置。开发者可以在发起购买前，查询该商品的优惠活动信息，在最终发起购买时，将优惠活动信息传递到华为应用内支付，最终将优惠活动信息展示给用户。</p>       <p>优惠促销规则：</p>       <ol>        <li>当用户同时符合优惠促销和推介促销时，优惠促销的优先级高于推介促销。</li>        <li>当用户历史上享受过优惠促销，仍具备享受推介促销的资格。</li>        <li>优惠促销无使用次数限制。</li>       </ol>       <div><div class="hw-editor-tip info"><div class="title">说明</div><div class="content">         <p>当前优惠促销涉及<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/iap-server-subscribe-offer-sign" target="_blank">生成优惠签名购买参数</a>处理，推荐具备服务器的开发者接入使用。</p>        </div></div></div></li>     </ul>     <ul>      <li id="li27291112195419"><strong>挽留促销（退订挽留促销，即将开放）</strong>       <p>适用于自动续期订阅商品的促销价格。开发者可以指定优惠持续时间、类型（免费试用、按周期扣费）和多次享受优惠的间隔（从优惠结束时间起算），可用于挽留即将取消订阅的用户。开发者配置自动续期订阅商品的挽留促销后，用户在订阅管理界面取消订阅且满足挽留促销使用条件时，会弹出挽留弹框，用户可以选择享受挽留促销并继续订阅或者仍要取消订阅。当用户选择享受挽留促销则下周期续期时挽留促销生效，享受挽留促销价。</p>       <p>挽留促销规则：</p>       <ol>        <li>用户使用了挽留促销，在续期前取消订阅，则挽留促销未生效。若该场景下用户恢复订阅，当前订阅周期内无法再次使用挽留促销，需完成续期后才可使用挽留促销。</li>        <li>开发者设置了挽留促销的使用间隔，当前时间与用户最近一次已享受挽留促销周期结束时间的间隔达到要求后，挽留促销才可再次使用。</li>        <li>挽留促销无使用次数限制，但以下场景不可使用。         <ol>          <li>用户正在享受或者下周期即将享受推介促销、优惠促销时，挽留促销不可使用。</li>          <li>用户正在享受或者下周期即将享受挽留促销时，挽留促销不可再次使用。</li>          <li>开发者设置了商品涨价且用户尚未同意涨价时，挽留促销不可使用。</li>         </ol></li>       </ol>       <p><span><img height="409.84722222222223" originheight="572" originwidth="1284" id="section060315441729">修改自动续期订阅商品<i class="anchor-icon anchor-icon-link" anchorid="section060315441729" tips="复制节点链接"></i></h2>         </div>    <div class="tiledSection">     <h3 id="section9790173617368" class="firsth2">调整自动续期订阅商品价格<i class="anchor-icon anchor-icon-link" anchorid="section9790173617368" tips="复制节点链接"></i></h3>          <p class="msonormal">开发者可以为每个自动续期订阅商品降低或提高价格，具体配置可参见<a href="https://developer.huawei.com/consumer/cn/doc/app/agc-help-modify-product-0000001146814541#section0980104712159" target="_blank">修改订阅类商品</a>。</p>     <ul>      <li class="msonormal">降低价格：对所有用户生效，包括已经订阅的用户和新订阅的用户。如果在距离下个续期日10天内调整价格， 则下次续期将按照现有价格再续订一个周期，然后再往后一个周期以新的价格续期。</li>      <li class="msonormal">提高价格：当提高自动续期订阅商品价格时，有两种策略可选。       <ul>        <li class="msonormal">为已订阅的用户保留原价，对新订阅的用户使用新的价格。</li>        <li class="msonormal">对已订阅的用户生效。IAP服务器会在下一个续期日提前10天通知已订阅的用户商品价格上涨以及新价格的收费日期。如果在距离下个续期日10天内调整价格， 则下次续期将按照现有价格再续订一个周期，然后在下个周期结束提前10天通知已订阅的用户。其他的限制还包括：         <ul>          <li>自动续期订阅商品新的价格需要已订阅用户的同意，如果用户未同意，其订阅不会自动续期，直到用户同意涨价。</li>          <li>在价格调整生效后将无法撤销。可以计划新的价格调整从而实现降价，但此前因价格提高而选择停止续订的用户需要重新订阅。</li>          <li>如果用户支付了促销价格的自动续期订阅商品且当前周期为以促销价格购买自动续期订阅商品的周期，则用户下一周期将按照原价进行订阅，再在下一个续费周期按照上述规则执行。</li>         </ul></li>       </ul></li>     </ul>    </div>    <div class="tiledSection">     <h3 id="section8821171943619">处理自动续期订阅商品下架<i class="anchor-icon anchor-icon-link" anchorid="section8821171943619" tips="复制节点链接"></i></h3>          <p>在处理自动续期订阅商品的下架时，需要考虑对用户的影响。</p>     <ul>      <li>允许存量用户续期，防止新用户订阅：开发者可以通过商品管理<a href="https://developer.huawei.com/consumer/cn/doc/app/set-digital-products-off-shelf-0000002107836877" target="_blank">下架商品</a>来防止新用户订阅，开发者仍需对存量用户提供订阅权益。</li>      <li>新/存量用户均不可用：开发者可以通过提前在商品管理中<a href="https://developer.huawei.com/consumer/cn/doc/app/delete-0000001959074905" target="_blank">删除数字商品</a>来防止新用户订阅以及存量用户续期。建议删除数字商品一段时间后再正式停止提供订阅权益，该时间需与其续费周期一致，保障存量用户体验到完整时限的商品服务。例如，对于周期为1个月的自动续期订阅商品，应该提前1个月将该商品删除，确保新用户在此期间无法订阅，且存量用户在停止提供订阅权益之前体验到完整时限权益，不会自动续期（自动购买下一期服务）。</li>     </ul>    </div>    <div class="tiledSection">     <h2 id="section1859111575215">接收订阅关键事件的通知<i class="anchor-icon anchor-icon-link" anchorid="section1859111575215" tips="复制节点链接"></i></h2>          <p>如果开发者提前配置了<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/iap-set-necessary-parameters#section22711950153313">通知接收地址</a>，用户购买自动续期订阅商品后，IAP服务器会在订阅场景的某些关键事件发生时调用此接口通知配置的服务器，具体请参见<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/iap-key-event-notifications" target="_blank">服务端关键事件通知</a>。主要涉及的场景如下：</p>     <ul>      <li>用户第一次订阅成功。</li>      <li>已过期的订阅自动续期成功。</li>      <li>用户主动恢复一个已过期的自动续期订阅商品。</li>      <li>用户调整为相同等级的自动续期订阅商品，如果订阅商品的周期相同，则新订阅立即生效。</li>      <li>用户调整为相同等级的自动续期订阅商品，如果订阅商品的周期不相同，则新订阅在下个续期日生效。</li>      <li>用户主动取消或App取消一个自动续期订阅商品，已经收费的服务仍然有效，但是后续续期会停止。</li>      <li>一个自动续期订阅商品成功续期。</li>      <li>一个到期的自动续期订阅商品进入<a href="/consumer/cn/doc/harmonyos-guides/iap-subscription-functions#section1656191811315">保留期</a>。</li>     </ul>     <p>若开发者服务器返回结果为非成功响应（请求返回的HTTP状态码不为200），将对本次关键事件的通知进行周期性重发。建议在服务器收到通知后立即返回成功响应，避免通知消息堆积。</p>    </div>    <div class="tiledSection">     <h2 id="section1474641116222">多角色应用接入订阅专项说明<i class="anchor-icon anchor-icon-link" anchorid="section1474641116222" tips="复制节点链接"></i></h2>          <p>用户购买订阅型商品时使用的账号是华为账号，一个华为账号在同一订阅商品上只能同时存在一笔订阅。如果应用存在多角色，可能会存在权益发放错乱的现象。针对这个场景建议的处理方案如下：</p>     <ol>      <li>当用户为角色A购买自动续期订阅商品后，该用户的其他角色只允许购买普通固定周期商品服务（开发者可以通过华为消耗型商品自行实现）。</li>      <li>当用户为角色A购买自动续期订阅商品后，再为角色B购买自动续期订阅商品时，开发者可以自行设计将角色A的剩余权益转移给角色B。</li>      <li>开发者可以配置多个自动续期订阅商品。如游戏不同区服对应不同角色，每个区服创建独立的自动续期订阅商品。</li>     </ol>    </div>   </div>   <div></div></div>
