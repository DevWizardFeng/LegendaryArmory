<h1 _ngcontent-ybm-c119="" class="doc-title ng-star-inserted" title="资源限定与访问"> 资源限定与访问 </h1>

<div _ngcontent-ybm-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;"> <div> <div class="tiledSection"><h2 id="资源限定词">资源限定词<i class="anchor-icon anchor-icon-link" anchorid="资源限定词" tips="复制节点链接"></i></h2><p>资源限定词可以由一个或多个表示应用场景或设备特征的限定词组合而成，包括屏幕密度等维度，限定词之间通过中划线（-）连接。开发者在<strong>resources</strong>目录下创建限定词文件时，需要掌握限定词文件的命名要求以及与限定词文件与设备状态的匹配规则。</p> </div>  <div class="tiledSection"><h2 id="资源限定词的命名要求">资源限定词的命名要求<i class="anchor-icon anchor-icon-link" anchorid="资源限定词的命名要求" tips="复制节点链接"></i></h2><ul><li><p>限定词的组合顺序：屏幕密度。开发者可以根据应用的使用场景和设备特征，选择其中的一类或几类限定词组成目录名称，顺序不可颠倒。</p>  </li><li><p>限定词的连接方式：限定词之间均采用中划线（-）连接。例如：res-dark-ldpi.json 。</p>  </li><li><p>限定词的取值范围：每类限定词的取值必须符合下表的条件，否则，将无法匹配目录中的资源文件，限定词大小写敏感。</p>  </li><li><p>限定词前缀：<strong>resources</strong>资源文件的资源限定词有前缀res，例如res-ldpi.json。</p>  </li><li><p>默认资源限定文件：<strong>resources</strong>资源文件的默认资源限定文件为res-defaults.json。</p>  </li><li><p>资源限定文件中不支持使用枚举格式的颜色来设置资源。</p>  </li></ul> <p><strong>表1</strong> 资源限定词</p>  <div class="tablenoborder"><div class="tbBox"><table class="layoutFixed idpTab"><thead><tr><th align="left" class="cellrowborder" id="mcps1.3.2.4.1.3.1.1" valign="top" width="50%">类型</th> <th align="left" class="cellrowborder" id="mcps1.3.2.4.1.3.1.2" valign="top" width="50%">含义与取值说明</th> </tr> </thead> <tbody><tr><td class="cellrowborder" valign="top" width="50%">屏幕密度</td> <td class="cellrowborder" valign="top" width="50%"><p>表示设备的屏幕密度（单位为dpi），取值如下：</p> <p>- ldpi：表示低密度屏幕（~120dpi）（0.75基准密度）</p> <p>- mdpi：表示中密度屏幕（~160dpi）（基准密度）</p> <p>- hdpi：表示高密度屏幕（~240dpi）（1.5基准密度）</p> <p>- xhdpi：表示加高密度屏幕（~320dpi）（2.0基准密度）</p> <p>- xxhdpi：表示超超高密度屏幕（~480dpi）（3.0基准密度）</p> <p>- xxxhdpi：表示超超超高密度屏幕（~640dpi）（4.0基准密度）</p> </td> </tr>  </tbody></table></div> </div> </div>  <div class="tiledSection"><h2 id="限定词与设备状态的匹配规则">限定词与设备状态的匹配规则<i class="anchor-icon anchor-icon-link" anchorid="限定词与设备状态的匹配规则" tips="复制节点链接"></i></h2><ul><li><p>在为设备匹配对应的资源文件时，限定词目录匹配的优先级从高到低依次为：MCC和MNC&gt; 横竖屏 &gt; 深色模式 &gt; 设备类型 &gt; 屏幕密度。在资源限定词目录均未匹配的情况下，则匹配默认资源限定文件。</p>  </li><li><p>如果限定词目录中包含资源限定词，则对应限定词的取值必须与当前的设备状态完全一致，该目录才能够参与设备的资源匹配。例如：资源限定文件res-hdpi.json与当前设备密度xhdpi无法匹配。</p>  </li></ul> </div>  <div class="tiledSection"><h2 id="引用js模块内resources资源">引用JS模块内resources资源<i class="anchor-icon anchor-icon-link" anchorid="引用js模块内resources资源" tips="复制节点链接"></i></h2><p>在应用开发的hml和js文件中使用$r的语法，可以对JS模块内的resources目录下的json资源进行格式化，获取相应的资源内容，该目录与pages目录同级，具体目录结构请参考<a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/js-framework-file">文件组织</a>。</p>  <div class="tablenoborder"><div class="tbBox"><table class="layoutFixed idpTab"><thead><tr><th
