<h1 _ngcontent-igi-c119="" class="doc-title ng-star-inserted" title="文本框"> 文本框 </h1>

<div _ngcontent-igi-c106="" auitextselectionexpansion="" class="markdown-body ng-star-inserted" style="position: relative;"> <div id="body0000001957012557"><p id="ZH-CN_TOPIC_0000001957012557__p102mcpsimp">文本框允许用户输入文本、选择文本。支持输入文本、数字或者混合格式的数据。单行文本输入类组件相关能力请参考 <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textinput" target="_blank">TextInput</a> 文档，多行场景请参考 <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textarea" target="_blank">TextArea</a> 文档。</p> <p class="msonormal" src="https://alliance-communityfile-drcn.dbankcdn.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20250619213502.53230563332296969441549590086316:50001231000000:2800:FD06E9D4BB5AE5017146CA2A904D84B0921E9F430F20D3C15B1C92752DA6AEEB.jpg" anchor-icon-link" anchorid="section76413581991" tips="复制节点链接"></i></h2><p id="ZH-CN_TOPIC_0000001957012557__p106mcpsimp"><strong>文本框应该有明确的标签或占位符，指示用户应该输入什么样的信息。</strong>对于敏感信息(如密码)，应该使用掩码字符来保护隐私，开发者可以在 <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textinput#inputtype枚举说明" target="_blank">InputType</a> 中查询到有关密码、邮件、号码等特定格式的样式类型。如果有特定的输入格式要求(如电话号码、邮箱地址等)，应该提供相应的验证和反馈。</p> <p id="ZH-CN_TOPIC_0000001957012557__p1324561720104"></p> <p id="ZH-CN_TOPIC_0000001957012557__p109mcpsimp"><strong>展示合理的文本提示反馈。</strong>在文本未输入前提供对应的提示文本，明确告知用户当前输入的信息类型、或者可填充内容。提示文本应当精简且直接点明主题，不要作内容运营或者诱导性提示。提示文本的样式规格可参考 <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textinput#placeholderfont" target="_blank">placeholder</a> 相关文档。</p> <p id="ZH-CN_TOPIC_0000001957012557__p16296118121013"></p> <p id="ZH-CN_TOPIC_0000001957012557__p112mcpsimp"><strong>合理展示和切换输入状态。</strong>当用户开始输入时，光标应该自动聚焦在文本框内，开发者可以配置 <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textinput#cancelbutton11" target="_blank">cancelButton</a> 来提供一键清空操作。对于无效的输入,应该给出明确的错误提示和解决方案，可以使用控件中的 <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textinput#showerror10" target="_blank">showError</a> 接口来进行配置。</p> <p id="ZH-CN_TOPIC_0000001957012557__p11129181961018"></p> <p id="ZH-CN_TOPIC_0000001957012557__p116mcpsimp"><strong>根据业务场景需要选择单行输入或是多行输入。</strong>实现多行输入有几种方式，一种是在 TextInput 中通过指定最大行数来提供有固定范围的行数约束，另外一种是使用 <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textarea" target="_blank">TextArea</a> 控件来实现多行文本输入，当输入的文本内容超过组件宽度时会自动换行显示，在 TextArea中如果开发者未设置高度，则控件无默认高度，跟随文本自适应内容，宽度未设置时，则默认撑满容器最大宽度。</p> <p id="ZH-CN_TOPIC_0000001957012557__p10977101991016"></p> <p id="ZH-CN_TOPIC_0000001957012557__p119mcpsimp"><strong>展示主题配置。</strong>在文本输入场景下会出现输入光标、文本选中、拖拽手柄等操作项，这些控件样式都会继承系统默认色彩风格。开发者可以通过配置 <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textinput#caretcolor" target="_blank">caretColor</a> 和 <a href="https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ts-basic-components-textinput#selectedbackgroundcolor10" target="_blank">selectBackgroundColor</a> 两个接口的颜色来配置针对应用自身的主题色样式。</p>
