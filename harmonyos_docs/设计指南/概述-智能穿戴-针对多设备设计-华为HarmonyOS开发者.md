id="section1632984120119">应用设计<i class="anchor-icon anchor-icon-link" anchorid="section1632984120119" tips="复制节点链接"></i></h2>         </div>    <p id="ZH-CN_TOPIC_0000002197410498__p1724110313164">在设计智能穿戴应用时，请考虑以下方法，这将帮助您提供优秀的用户体验。智能穿戴设备相比其他大屏幕设备具有独特的应用场景，在设计时需要结合产品小而轻的特征做场景适应性分析。</p>    <div class="tiledSection">     <h3 id="section164191521125813" class="firsth2">保证基础体验<i class="anchor-icon anchor-icon-link" anchorid="section164191521125813" tips="复制节点链接"></i></h3>         </div>    <ul id="ZH-CN_TOPIC_0000002197410498__ul324118331613">     <li id="ZH-CN_TOPIC_0000002197410498__li1124143141620"><strong>即时通知：</strong>智能穿戴的天然跟随属性，受时间、空间影响因素较小，有利于即时提醒和告知与用户相关的消息。另外需要考虑部分场景下重复、无效的通知对用户造成的干扰，根据用户真实的佩戴情况做通知处理。</li>     <li id="ZH-CN_TOPIC_0000002197410498__li624116361615"><strong>轻量交互：</strong>智能穿戴手表小而轻，应该专注重要且主要的功能体验。同一应用内容，智能穿戴应该发挥移动优势，做其他大屏幕设备的补充和延伸，而非替代者。在具体应用设计中，需要根据智能穿戴手表的尺寸大小、使用场景做轻量化定制，并满足当前设备应用界面易浏览、交互流程易操作。</li>    </ul>    <div class="tiledSection">     <h3 id="section13939154135814">设计应用体验<i class="anchor-icon anchor-icon-link" anchorid="section13939154135814" tips="复制节点链接"></i></h3>         </div>    <ul id="ZH-CN_TOPIC_0000002197410498__ul824193201613">     <li id="ZH-CN_TOPIC_0000002197410498__li52416311614"><strong>使用系统控件：</strong>利用系统提供的导航栏、按钮及导航点等标准控件，在保证良好基础体验的同时，减少设计和开发的工作量。必要时自定义控件的样式和大小以体现自己的品牌特征。请参阅控件-<a href="https://developer.huawei.com/consumer/cn/doc/design-guides/general_overview-0000001929599380">概览</a>。</li>     <li id="ZH-CN_TOPIC_0000002197410498__li62411311169"><strong>使用合适的应用架构：</strong>根据业务的特点采用合适的架构。例如，单一功能类应用采用页面横向或纵向平铺的平级架构，以确保主要功能在首屏展示。请参阅<a href="https://developer.huawei.com/consumer/cn/doc/design-guides/application-architecture-0000002202799801#section4983154519514">应用架构</a>。</li>     <li id="ZH-CN_TOPIC_0000002197410498__li82414313168"><strong>保证页面正常显示：</strong>合理根据智能穿戴的屏幕形状进行界面设计，确保界面正常显示，不影响用户的使用。</li>     <li id="ZH-CN_TOPIC_0000002197410498__li112417321610"><strong>充分利用表冠和按键交互：</strong>除屏幕手势交互外，表冠和按键交互能够帮助用户在需要盲操场景下快速完成特定操作。例如单击表冠快速开始/暂停运动等。请参阅<a href="https://developer.huawei.com/consumer/cn/doc/design-guides/human-machine-interaction-0000002167648022#section02103311681">表冠</a>。</li>     <li id="ZH-CN_TOPIC_0000002197410498__li524117311167"><strong>考虑单手操作：</strong>在特殊情况下，用户不得不单手对穿戴设备进行操作，应用在设计界面时，应考虑将核心功能适配焦点响应，使用户能够单手操作。请参阅<a href="https://developer.huawei.com/consumer/cn/doc/design-guides/human-machine-interaction-0000002167648022#section239618496133">智慧手势</a>。</li>     <li id="ZH-CN_TOPIC_0000002197410498__li9241332164"><strong>跨设备体验连续：</strong>手机与手表在部分关联任务上存在延续性。任务进行时，界面需要实时引导用户移至相应配对设备进行相关操作。在整个任务延续过程中，多端侧状态的同步和显示要尽量保持一致或关联性。例如添加 eSIM 账号时，可通过手表向手机发送指令，打开界面。</li>    </ul>    <div class="tiledSection">     <h3 id="section3580125015213">支持系统特性<i class="anchor-icon anchor-icon-link" anchorid="section3580125015213" tips="复制节点链接"></i></h3>         </div>    <ul
